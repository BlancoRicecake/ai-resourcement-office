# 디자인 툴·조회 도구 연동 (선택)

이 번들은 **자체 MCP 서버를 포함하지 않는다.** 순수 마크다운(지시문 + 지식 노드 + 스킬 + 메모리)만으로 전체 워크플로가 동작한다. 아래는 **호스트 런타임 쪽**에 연결하면 자문가가 화면을 더 잘 조회할 수 있는 선택적 연동이다. 연결되면 자문가는 그 능력을 도구 연결 스코프(`AGENTS.md`) 안에서만 쓰고, 없으면 스크린샷 → 화면 텍스트 묘사 순으로 대체한다.

입력 확보 우선순위: **① 연결된 디자인 툴·라이브 화면 직접 조회 → ② 스크린샷 → ③ 텍스트 묘사.**

## 1. Figma MCP (공식) — 디자인 파일 리뷰

Figma 공식 MCP를 호스트에 연결하면 Figma 파일·프레임을 직접 조회해 리뷰할 수 있다. 정확한 설치 절차는 **Figma 공식 MCP 문서**를 따른다(버전에 따라 커맨드가 바뀔 수 있어 여기서 단정하지 않는다). Claude Code 기준 일반형 예시:

```bash
# 일반형 예시 — 실제 서버/인자는 Figma 공식 MCP 문서 확인
claude mcp add figma -- <figma-mcp-실행-커맨드>
```

- 리뷰 시 대상 프레임/노드만 지정해 조회한다. 파일 전체를 통째로 노출하지 않는다.

## 2. Pencil MCP — 로컬 `.pen` 디자인 파일 리뷰

`.pen` 디자인 파일을 쓰는 경우 Pencil MCP를 연결하면 로컬 디자인 파일을 조회해 리뷰할 수 있다. `.pen`은 전용 도구로만 열리므로, 연결된 Pencil MCP의 조회 범위 안에서 프레임을 읽어 진단한다.

## 3. 브라우저 확장·멀티모달 (Claude in Chrome 등) — 라이브 사이트 리뷰

브라우저 자동화/확장(예: Claude in Chrome)이 연결되면 라이브 사이트를 직접 열람하며 리뷰한다(실제 렌더·인터랙션 확인에 유리). 연결이 없으면 사용자에게 스크린샷이나 화면 묘사를 요청한다.

## 권한 주의

- 디자인 툴·브라우저에는 **리뷰에 필요한 최소 파일/프레임/페이지만** 노출한다.
- **미공개 디자인·인증 뒤 화면**을 연결할 때는 공유 범위를 사용자가 결정한다 — 그 콘텐츠가 호스트 런타임·모델 제공자에게 전달됨을 알린다. 민감 화면은 마스킹·발췌 후 조회를 권한다.
- 자문가는 미공개 화면·내부 지표를 `memory/`에 기록하지 않는다.
