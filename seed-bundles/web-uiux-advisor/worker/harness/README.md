# Harness: UI·UX 자문가 구동 방법

이 워커는 순수 마크다운 번들이다. 빌드·설치·API 키가 필요 없다. 판단은 이 번들을 구동하는 호스트 코딩 에이전트/LLM이 맡고, 화면 수집(URL fetch)·스크린샷 해석은 호스트 런타임의 능력을 그대로 쓴다.

## 방법 1: 코딩 에이전트 런타임(Claude Code / Codex 등)에서 폴더로 구동

1. 이 번들 폴더를 에이전트 작업 디렉토리로 연다.
2. 런타임이 `AGENTS.md`의 로드 순서를 따르게 한다: `worker/agent.md` → `memory/`(표준 4파일 + REVIEWS.md, `knowledge/`는 INDEX만) → `worker/skills/`.
3. 두 시나리오 중 하나로 요청한다.
   - 기획 자문: 서비스 설명·기획안을 주고 설계 방향·레퍼런스를 요청한다.
   - 화면 리뷰: URL·스크린샷·HTML·텍스트 묘사를 주고 진단을 요청한다.
4. 자문가가 인테이크 질문(서비스·타깃·목표·플랫폼)을 먼저 하고, 답이 모이면 리뷰 리포트를 낸다.

## 방법 2: LLM 채팅(수동)에서 구동

1. `worker/agent.md` 전문을 시스템 지시문(또는 첫 메시지)으로 붙여넣는다.
2. 관련 지식이 필요하면 `memory/knowledge/INDEX.md`와 해당 노드 파일을 함께 붙여넣는다.
3. 화면을 텍스트로 묘사하거나 렌더된 HTML을 붙여넣어 리뷰를 요청한다(채팅 환경은 URL/스크린샷 해석이 제한될 수 있으므로 텍스트 묘사 폴백을 권장).

## 방법 3: Claude Code named agent로 구동 (선택)

`worker/agent.md`를 named agent 지시문으로 등록하면 프로젝트 안에서 `@ui-ux-advisor`처럼 호출할 수 있다. 이는 여러 설치 경로 중 하나이며 기본 형태는 아니다.

## 권한 원칙

- 워커에게 필요한 것은 이 번들 폴더의 `memory/` 읽기/쓰기 권한과, (리뷰 대상이 URL이면) 호스트 런타임의 네트워크 fetch 능력뿐이다.
- URL을 열 수 없거나 스크린샷을 해석할 수 없는 환경이면, 사용자에게 화면의 텍스트 묘사나 렌더된 HTML을 요청하는 흐름으로 대체한다.
- 결정론 계산 도구(app/)는 이 번들에 없다 — 별도 실행 권한이 필요 없다.
