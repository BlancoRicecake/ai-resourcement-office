# worker/skills — 업무 단계별 규칙

UI·UX 자문가가 리뷰·자문 단계에서 따르는 규칙 문서다. `worker/agent.md`의 워크플로(③)가 각 단계에서 이 스킬들을 호출한다.

| 스킬 | 언제 쓰나 | 핵심 |
| --- | --- | --- |
| `ui-review-intake.md` | 요청이 들어왔는데 서비스·타깃·목표가 불명확할 때 | SERVICE/TARGET/GOAL/PLATFORM/CONSTRAINTS 질문지. 필수 3항목 없으면 진단 시작 금지. |
| `landing-page-review.md` | 랜딩·홈·마케팅 페이지 리뷰 | Above-fold 5요소/Mid/Bottom 섹션 체크, 4레이아웃 타입, 공통 실패 패턴. |
| `evidence-based-audit.md` | 모든 Finding을 쓸 때 | 주장→증거 연결, "검증 필요" 표기, 심각도 0~4 산정. |
| `feedback-handoff-prompt.md` | 리포트를 마무리하고 구현자에게 넘길 때 | 리뷰 결과를 복붙 가능한 구현 지시 프롬프트로 변환(수정 지시+보존 항목+제약+완료 기준). |

## 런타임 학습 절차

이 스킬들은 seed(번들 동봉)다. 리뷰를 반복하며 재사용 가능한 절차나 체크 항목이 확인되면(예: "특정 업종 랜딩에서 반복되는 미스"), 스킬을 새로 만들기보다 관련 스킬을 개선할 **`메모리 업데이트 후보`**로 제안한다(자동 확정 금지). 일반화 가능한 원칙은 `memory/knowledge/`의 노드로 승격하고, 절차성 개선은 여기 스킬 문서의 갱신 후보로 정리한다. 반영은 사용자 승인 후에 한다.

일부 스킬의 워크플로 구조는 github.com/MengTo/Skills (MIT, Meng To)에서 영감을 받아 재구성했으며, 각 스킬 파일 하단에 원본명을 표기했다.
