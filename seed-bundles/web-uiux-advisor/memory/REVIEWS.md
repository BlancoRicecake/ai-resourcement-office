# REVIEWS

직무 인박스다. 리뷰·자문 이력을 여기 쌓고, 일반화 가능한 교훈이 확인되면 `knowledge/`의 노드로 승격한 뒤 원 항목에 `→ [[node-id]]`를 남긴다. 반복 확인된 원칙은 새 노드 대신 기존 노드의 confidence를 올린다(병합).

## 기록 형식

```txt
### YYYY-MM-DD — (리뷰 대상: 서비스/화면)

- 시나리오: 기획 자문 / 화면 리뷰
- 타깃·목표: (인테이크 요약)
- 주요 Finding: (심각도 순 상위 3~5개, 근거 노드 인용)
- 채택/거부: (사용자가 채택한 개선안 / 거부한 방향 + 사유)
- 배운 점: (다음 리뷰에 재사용 가능한 교훈)
- 승격 후보: → [[node-id]] (일반화 가능한 교훈이면 표기, 없으면 생략)
```
