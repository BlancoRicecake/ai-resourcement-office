Jon Yablonski가 정리한 UX 법칙 모음에서 웹 리뷰에 자주 쓰는 핵심을 요약한다. 각 법칙은 개별 노드로 깊게 다룬다.

- **Fitts의 법칙** — 타깃 거리·크기와 클릭 시간. CTA는 크고 가깝게, 위험한 액션은 작고 멀게. [[fitts-law]]
- **Hick의 법칙** — 선택지가 많을수록 결정이 느려진다. 그룹핑·단계화로 축소. [[hicks-law]]
- **Jakob의 법칙** — 익숙한 패턴을 따르면 학습비용이 0에 가깝다. [[jakobs-law]]
- **Miller의 법칙** — 단기기억 처리 단위 제한. 정보는 청크로 묶는다.
- **Postel의 법칙** — 입력은 관대하게 받고 출력(에러)은 명확하게.
- **Peak-End 규칙** — 절정·마지막 순간이 경험 평가를 좌우한다. [[peak-end-rule]]
- **Aesthetic-Usability Effect** — 미적 완성도가 사용성 인식에 기여한다. [[aesthetic-usability-effect]]
- **Doherty 임계** — 응답 400ms 이내에서 몰입이 급증한다. [[doherty-threshold]]
- **Von Restorff / Tesler** — 주변과 다른 요소가 기억에 남는다 / 복잡성은 없앨 수 없고 누가 감당할지의 문제다.

법칙은 규칙이 아니라 판단의 렌즈다. 서비스 목표에 비춰 적용한다.
