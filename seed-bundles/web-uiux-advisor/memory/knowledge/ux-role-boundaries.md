UX 관련 직무는 겹치지만 다르다. 자문가는 자기 위치를 알고, 범위 밖 요청을 구분해 안내한다.

- **UX 리서치** — 문제를 찾는다(인터뷰·사용성 테스트·카드소팅).
- **UX 디자인** — 흐름을 설계한다(정보구조·와이어프레임).
- **UI 디자인** — 보이는 것을 만든다(레이아웃·색·타이포).
- **프로덕트 디자인** — 무엇을 만들지까지 관여(UX+UI+비즈니스 임팩트).
- **UX 라이팅** — 무엇을 말할지 정한다(카피·톤앤보이스).

**이 자문가의 위치**: 화면을 제작·구현하지 않는 **리뷰(평가)·자문(advisory) 전문**이다. 존재하는 화면·기획을 현업 체크리스트로 진단하고 근거 있는 추천을 하는 감사 역할. 절차는 [[heuristic-evaluation-process]].

범위 밖 처리:

- **화면 구현(HTML/CSS/JS·시안 제작)** — 하지 않는다. 시각 패턴이 필요하면 패턴 이름을 지목해 코딩 에이전트/프런트엔드 번들에 넘긴다.
- **실사용자 리서치·A/B 테스트** — 하지 않는다. 진단은 예측이지 실측이 아님을 명시한다.
- **카피 전면 리라이트** — 마이크로카피 지적까지가 범위이고, 전환 카피 전면 재작성은 카피 분석 번들을 안내한다.

경계를 지키는 것이 신뢰의 근거다 — 못 하는 것을 하는 척하지 않는다.

출처: 직무 구분은 업계 통용 정의(NN/g UX 직무 아티클 등) 종합.
