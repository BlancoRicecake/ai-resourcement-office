개선안·기획 자문에서 "이렇게 하라"의 근거로 실제 프로덕트를 인용한다. 갤러리마다 용도와 함정이 다르다.

- **Mobbin** — 실제 출시된 프로덕트의 UI 패턴·플로우 아카이브. 컴포넌트·플로우 레퍼런스에 최적(익숙한 패턴 확인용). [[jakobs-law]]
- **Land-book** — 업종별 랜딩페이지 갤러리. 랜딩 구조 벤치마크.
- **SaaS Landing Page / Saaspo / Lapa Ninja** — SaaS 특화, 폰트·팔레트·스택 필터링.
- **Awwwards** — 스타일 다양성 최고지만 **심미성 중심 심사**다. usability와 동일시하지 않는다(캐비어트 필수). 시각 영감으로만 인용한다.

리뷰 관점:

- 근거 우선순위는 지식 노드 → 갤러리 → 디자인 시스템이다. 갤러리는 "실물 사례" 근거지 규범은 아니다.
- 유사 서비스 3~5개를 비교해 "관습"을 확인한다 — 다수가 같은 방식이면 사용자 기대다.
- 권위 있는 규범은 디자인 시스템에서 가져온다. [[design-systems-authority]]

인용 시 "무엇을 왜" 참고하는지 명시한다(막연한 "OO처럼" 금지).

출처: Mobbin / Land-book / Awwwards / Saaspo 공개 갤러리.
