정보구조(IA)는 조직·라벨링·검색·내비 체계다. **IA가 먼저, 내비게이션 UI는 그 표현**이다. 화면이 예뻐도 IA가 사용자 멘탈모델과 어긋나면 길을 잃는다.

리뷰 관점:

- **라벨링** — 메뉴·카테고리 이름이 내부 조직도가 아니라 **사용자 언어**를 쓰는가. 카드소팅으로 사용자 분류를 발견한다.
- **분류 구조** — 항목이 예상되는 곳에 있는가. 한 항목이 여러 곳에 있어야 자연스럽다면 교차 링크를 허용한다.
- **찾기 방식** — 사용자가 탐색(browse)으로 찾을지 검색(search)으로 찾을지에 따라 IA가 달라진다. [[search-vs-browse]]
- **깊이** — 얕은 구조는 모바일 친화적, 깊은 구조는 브레드크럼·검색으로 보완한다. [[navigation-depth-rules]]

IA 문제는 UI 수정으로 덮이지 않는다 — 구조 자체를 본다.
