마이크로카피는 버튼·레이블·에러·빈 상태·툴팁의 짧은 문구다. 작지만 마찰과 신뢰를 좌우한다. 자문가는 마이크로카피 지적까지 하고, 전면 카피 리라이트는 범위 밖으로 넘긴다.

지침:

- **버튼은 행동+결과** — "제출" 대신 "무료로 시작하기", "확인" 대신 "예약 확정". 모호한 라벨은 미스.
- **에러는 원인+해결책** — "오류 발생"이 아니라 "이메일 형식이 올바르지 않습니다. 예: name@example.com". 필드 바로 아래에. [[form-ux-best-practices]]
- **사용자를 탓하지 않는다** — "잘못 입력했습니다"보다 "다시 확인해 주세요". 빈 상태·무결과도 격려하며 다음 행동으로. [[empty-state-onboarding]]
- **시스템 용어 금지** — "null", "세션 만료 코드 419" 같은 내부 용어를 노출하지 않는다(Nielsen 2번).
- **일관된 어휘·톤** — 같은 개념을 화면마다 다른 단어로 부르지 않는다. 브랜드 톤(존댓말 등)을 지킨다.

리뷰 관점:

- 카피 지적도 근거를 단다("행동+결과 원칙 위반"). 확인 못 한 문구는 추측하지 않는다.
- 저작권 있는 원문을 그대로 옮기지 않고 개선 방향만 제시한다.
