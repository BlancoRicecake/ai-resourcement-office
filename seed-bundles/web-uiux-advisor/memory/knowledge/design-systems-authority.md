규범이 필요할 때(컴포넌트 동작·간격·접근성) 권위 있는 디자인 시스템을 인용한다. 각각 강한 영역이 다르다.

- **Material Design 3 (Google)** — 크로스플랫폼 컴포넌트·모션·컬러 토큰 표준. 안드로이드·웹 일반.
- **Apple HIG** — iOS/macOS 네이티브 인터랙션 관습의 최종 권위. 터치 타깃·제스처.
- **Shopify Polaris** — 어드민·B2B 대시보드 설계 표준.
- **IBM Carbon** — 엔터프라이즈·데이터 밀도형 토큰 시스템.
- **Atlassian Design System** — 협업툴 특유의 토큰 우선 아키텍처.
- **Astryx (Meta)** — React + StyleX 기반 오픈소스 디자인 시스템. 접근성 내장 컴포넌트(160+)·테마/다크모드·에이전트 친화 구조. **React 웹 컴포넌트 구현 관습·접근성 컴포넌트·테마링**의 참조 규범(웹 전용 자문 범위에 부합).

리뷰 관점:

- 플랫폼에 맞는 시스템을 근거로 든다(iOS 앱 리뷰에 Material 규범을 들이대지 않는다).
- 디자인 시스템은 갤러리보다 강한 근거다 — "권장값"의 출처가 된다(예: 터치 타깃 44/48). [[reference-galleries]]
- 시스템을 맹목 적용하지 않는다 — 서비스 맥락과 충돌하면 이유를 대고 조정한다.

인용은 규범의 출처로만 쓰고, 저작권 있는 원문·에셋을 복제하지 않는다.
