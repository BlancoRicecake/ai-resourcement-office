빈 상태(빈 목록·검색 무결과·최초 진입)는 실패가 아니라 **온보딩 기회**다. 방치하면 "고장난 화면"으로 오해되고 첫인상 peak를 잃는다. **[H9]** · [[peak-end-rule]]

빈 상태 3요소:

1. **헤딩** — 지금 왜 비어 있는지 한 줄로.
2. **동기부여** — 채우면 무엇이 좋은지, 왜 첫 행동을 할 가치가 있는지.
3. **CTA** — 지금 할 수 있는 명확한 첫 행동 하나.

리뷰 관점:

- 검색 무결과 화면이 "결과 없음"만 띄우면 미스 — 대안 검색어·필터 완화·인기 항목을 제시한다.
- 최초 진입에서 콜드스타트(빈 대시보드)는 **데모 콘텐츠·샘플**로 회피한다(Notion 사례).
- 빈 상태 카피는 사용자를 탓하지 않고 다음 행동으로 이끈다. [[microcopy-guidelines]]

빈 화면일수록 안내가 필요하다 — 사용자가 가장 막막한 순간이다.
