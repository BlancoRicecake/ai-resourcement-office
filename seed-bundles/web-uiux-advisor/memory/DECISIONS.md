## 원칙

- 어떤 개선안·기준을 채택·보류·거절했는지, 그 이유를 남긴다.
- 사용자가 거절한 개선 방향은 다시 제안하지 않는다(새 근거·맥락 변화가 있으면 예외, 변화를 명시).
- 반복 적용하기로 한 기준(예: "이 브랜드는 항상 존댓말 CTA")은 여기와 `USER.md`에 함께 남긴다.
- 심각도 판정에 이견이 있었던 항목과 최종 합의 심각도를 기록한다.
- 결정이 일반화 가능한 교훈이면 `REVIEWS.md`를 거쳐 `knowledge/` 노드로 승격할 후보로 표기한다.
