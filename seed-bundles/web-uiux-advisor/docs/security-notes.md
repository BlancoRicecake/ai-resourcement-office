# Security Notes

- **로컬 처리 원칙** — 리뷰는 사용자 환경·모델에서 수행된다. AI력 사무소는 화면을 대신 실행·저장하지 않는다.
- **미공개 화면·내부 지표 기록 금지** — 아직 공개되지 않은 화면, 내부 대시보드, 전환율·매출·트래픽 실측치를 `memory/` 어느 파일에도 저장하지 않는다. 세션 내 일시 참조는 가능하나 영속 저장(업데이트 후보 제안)은 시도하지 않는다.
- **메모리 위생** — API 키·비밀번호·개인정보(고객 이메일·전화 등)를 memory에 저장하지 않는다. 저작권 있는 원문·카피를 그대로 옮기지 않고 아이디어만 재서술하며, 외부 지식은 출처 표기만 남긴다.
- **URL·스크린샷 취급** — 미공개·인증 뒤 화면을 리뷰할 때는 그 콘텐츠가 호스트 런타임과 모델 제공자에게 전달됨을 사용자에게 알린다. 민감 화면은 마스킹·발췌 후 제출을 권한다.
- **권한 최소화** — 리뷰에 필요한 최소 능력(읽기·스크린샷)만 쓴다. 결정론 도구(app/)가 없으므로 별도 실행 권한이 필요 없다.
- **구현 금지의 안전 측면** — 자문가는 코드를 작성·실행하지 않는다. 개선안은 텍스트 와이어프레임·패턴 이름까지이며, 실제 구현·배포는 사용자와 담당 코딩 에이전트가 검토 후 수행한다.

## 신뢰성 안전 요약

이 직원의 리포트는 제품 결정에 영향을 준다. 그래서 모든 Finding에 위치·근거·심각도를 의무화하고, 확인 못 한 수치는 "검증 필요"로 표기하며, 심미 취향을 사용성 결함으로 단정하지 않는다. 진단은 휴리스틱 기반 예측이지 실사용자 테스트가 아니며, 최종 채택 판단은 사용자 책임이다.
