# [예시 실행 결과] TeamFit 랜딩페이지·가입 폼 UI/UX 리뷰 리포트

> 이 문서는 `examples/input/screen-description.md`를 입력으로, `worker/agent.md`의 페르소나·워크플로와 `memory/knowledge/` 지식 노드를 적용해 UI·UX 자문가가 실제로 작성한 리뷰 리포트다. 화면을 구현하지 않고 진단·개선안까지만 다룬다.

## 인테이크 확인 (진단 전 확보)

- 서비스: 중소기업용 근태·연차 관리 SaaS(TeamFit).
- 타깃: 5~50인 회사의 인사 담당자(비개발자, 엑셀 익숙). 결정자는 대표·총무.
- 핵심 목표: 랜딩 방문자 → "14일 무료 체험" 가입.
- 플랫폼: 데스크톱 웹 + 모바일웹.
- 대상 시장: 인테이크에서 명시 미확인 → **보편 원칙만 적용**(한국 등 특정 시장 관습 미적용). 시장이 확인되면 해당 관습을 추가 검토.
- 근거 유형: 화면의 텍스트 묘사(스크린샷 아님) → 색값·동작·크기 일부는 "검증 필요"로 표기함.

## Executive Summary (상위 5)

1. **히어로만 봐선 무엇을 하는 서비스인지 알 수 없다.** 폴드 위에 "근태·연차 관리"라는 가치제안이 없고, 실제 설명은 스크롤 1~2번 아래에 있다. 전환의 첫 관문이 비어 있다. (심각도 4)
2. **폴드 위 1차 CTA가 3개로 경쟁한다.** "무료 체험 / 데모 / 요금제"가 동급 크기라 결정이 분산된다. 핵심 목표(무료 체험)가 묻힌다. (심각도 3)
3. **가입 폼이 이탈을 부른다.** 레이블이 플레이스홀더로만 있고, 비밀번호 규칙을 제출 후에야 알려 준다. 전환 직전 단계라 영향이 크다. (심각도 3)
4. **접근성 하한 위반 2건.** 히어로 서브텍스트 저대비(추정 약 2:1)와 모바일 하단 아이콘 20px(24px 미만). 일부 사용자에게 읽기·조작 차단. (심각도 3~4)
5. **신뢰 신호(고객 후기)가 결정 지점에서 멀다.** 후기가 페이지 맨 아래 푸터 위에 매몰돼, 결정하는 순간에 신뢰 근거가 없다. (심각도 2)

이 서비스의 강점 방향: 기능 자체는 명확(출퇴근·연차·근무표)하고 타깃도 뚜렷하다. 문제는 **그 강점이 폴드 위에서 전달되지 않는다**는 것 하나로 수렴한다.

## Findings (근거·심각도)

### F1 — 히어로에 가치제안 부재 · 무엇을 파는지 모름
- **위치**: 히어로(폴드 위) 헤드라인·서브헤드.
- **관찰**: 헤드라인 "일하는 방식의 새로운 기준", 서브헤드 "더 나은 팀을 만드세요". 둘 다 슬로건이고, "근태·연차 관리 SaaS"라는 설명은 스크롤 1~2번 아래에서야 등장.
- **근거**: 조회 시간의 약 80%가 폴드 위에 집중되는데 핵심 가치제안이 폴드 아래에 있음 — [[above-the-fold-priority]]. 랜딩 Above-fold 5요소 중 "가치제안 헤드라인"이 슬로건으로 대체됨 — [[landing-page-review-checklist]]. 헤드라인만 읽어도 무엇을 파는지 알아야 함.
- **심각도**: 4 (재앙적) — 무엇을 하는지 모르면 나머지 설계가 무의미. 빈도(모든 방문자)·영향(전환 차단)·지속성(매 방문) 모두 최대.

### F2 — 폴드 위 1차 CTA 3개 경쟁
- **위치**: 히어로 버튼 영역("무료 체험 시작 / 데모 영상 보기 / 요금제 확인", 동급 크기).
- **관찰**: 세 버튼이 비슷한 크기로 나란히 있어 무엇이 1차 행동인지 위계가 없음.
- **근거**: 선택지가 늘면 결정이 느려짐 — [[hicks-law]]. 랜딩 공통 실패 패턴 "CTA 과다·경쟁" — [[landing-page-review-checklist]]. 핵심 목표는 무료 체험 하나.
- **심각도**: 3 (심각).

### F3 — 제품 실물 부재 (일러스트만)
- **위치**: 히어로 우측 비주얼.
- **관찰**: 웃는 사람 일러스트만 있고 실제 제품 화면(대시보드·근무표)이 보이지 않음.
- **근거**: Above-fold 5요소 중 "제품 비주얼/데모"가 추상 일러스트로 대체됨 — [[landing-page-review-checklist]]. 비개발자 인사 담당자에게 "이게 어떻게 생겼나"는 신뢰·이해의 핵심.
- **심각도**: 2 (경미하나 전환에 영향).

### F4 — 상단 메뉴가 hover에만 의존
- **위치**: 상단 바 "기능"·"요금제" 드롭다운.
- **관찰**: 마우스를 올려야 펼쳐지고 클릭은 무반응. 모바일에선 hover가 없어 접근 불가.
- **근거**: hover 의존은 터치 디바이스에서 접근 불가 — [[thumb-zone-mobile]]. 관습(클릭 시 열림/이동) 위반 — [[jakobs-law]]. 모바일웹이 주요 플랫폼 중 하나라 영향 큼.
- **심각도**: 3 (심각) — 모바일 사용자에게 하위 메뉴 접근 차단.

### F5 — 히어로 서브텍스트 저대비
- **위치**: 히어로 서브헤드(연한 회색 #b3b3b3, 흰 배경).
- **관찰**: 연회색 본문. 추정 대비 약 2:1 수준.
- **근거**: WCAG 1.4.3 본문 대비 하한 4.5:1 위반 추정 — [[wcag-aa-checklist]]. 저대비는 가장 흔한 접근성 실패.
- **심각도**: 3 (심각). **검증 필요**: 실제 hex/대비비는 개발자도구로 실측 확인 요망(목표 4.5:1).

### F6 — 모바일 하단 아이콘 타깃 과소·과밀
- **위치**: 모바일웹 하단 고정 아이콘 바(홈·검색·문의·계정).
- **관찰**: 아이콘 약 20px, 서로 바짝 붙어 옆 아이콘이 자주 눌림.
- **근거**: WCAG 2.5.8 최소 24×24px 미달, 관행 권장 44/48 크게 미달, 인접 간격 부족 — [[touch-target-size]]. 엄지존 하단 배치는 맞으나 크기·간격이 오조작을 유발 — [[thumb-zone-mobile]].
- **심각도**: 3 (심각). **검증 필요**: 실제 px·간격 실측.

### F7 — 가입 폼: 플레이스홀더 레이블 + 사후 검증 + 규칙 미안내
- **위치**: 가입 폼(이름·회사 이메일·비밀번호·회사명).
- **관찰**: (a) 레이블이 없고 플레이스홀더로만 안내 → 입력 시작하면 사라짐. (b) 비밀번호 규칙 안내 없음. (c) 제출 후에만 상단에 에러.
- **근거**: 레이블은 필드 위, 검증은 blur 시점, 에러는 필드 바로 아래 원인+해결책 — [[form-ux-best-practices]]. 에러·안내 카피가 사후·상단이라 마이크로카피 원칙 위반 — [[microcopy-guidelines]]. 전환 직전 단계라 이탈 영향 큼.
- **심각도**: 3 (심각).

### F8 — 검색 무결과 빈 상태 방치
- **위치**: 고객사례 검색 결과 없음 화면(모바일).
- **관찰**: 결과가 없을 때 화면이 텅 비고 "검색 결과 없음"조차 없음.
- **근거**: 빈 상태 3요소(헤딩+동기부여+CTA) 전무 — [[empty-state-onboarding]]. 무결과 시 대안 제시 부재 — [[search-vs-browse]]. "고장난 화면"으로 오해됨.
- **심각도**: 2 (경미).

### F9 — 신뢰 신호(후기) 하단 매몰
- **위치**: 고객 후기 2개(페이지 맨 아래, 푸터 위).
- **관찰**: 후기가 결정 지점(폴드·요금제 근처)에서 멀리 떨어짐.
- **근거**: 랜딩 공통 실패 "증거 하단 매몰" — [[landing-page-review-checklist]]. 사회적 증거는 결정 지점 근처에 있어야 신뢰로 작동.
- **심각도**: 2 (경미).

### F10 — 기능 설명이 혜택 아닌 라벨 나열
- **위치**: "왜 TeamFit인가" 기능 3개("간편한 출퇴근" 등).
- **관찰**: 아이콘+짧은 제목뿐, 구체적으로 무엇을 해 주는지 없음.
- **근거**: 명확성 — 헤드라인·기능 설명이 "그래서 담당자가 얻는 것"으로 환원돼야 함(Nielsen 2·8) — [[nielsen-10-heuristics]].
- **심각도**: 1 (코스메틱).

## 개선안 (Before/After + 텍스트 와이어프레임)

### F1·F2·F3 — 히어로 재구성 (최우선)

**Before (현재 폴드 위)**
```
[로고]                 기능 요금제 고객사례 블로그 회사소개  로그인 [무료체험]
        일하는 방식의 새로운 기준.
        더 나은 팀을 만드세요.            (연회색, 잘 안 보임)
   [무료 체험 시작] [데모 영상 보기] [요금제 확인]   [웃는 사람 일러스트]
```

**After (텍스트 와이어프레임)**
```
[로고]                 기능 요금제 고객사례 블로그   로그인 [14일 무료 체험]
   ┌─────────────────────────────┬──────────────────────────┐
   │ H1: 출퇴근·연차·근무표를       │                          │
   │     한곳에서. 5~50인 팀을 위한 │   [ 실제 대시보드 스크린샷 ] │
   │     근태·연차 관리 SaaS       │   (근무표/연차 승인 화면)   │
   │ 서브헤드(충분한 대비):        │                          │
   │  엑셀 없이, 담당자 혼자서 관리. │                          │
   │ [14일 무료 체험 시작] ← 1차만 강조                        │
   │  데모 영상 보기(텍스트 링크, 2차)  요금제 →(하단 앵커)      │
   │ ★★★★★ 300+ 팀이 사용 중 (신뢰 신호)                     │
   └─────────────────────────────┴──────────────────────────┘
```
- 헤드라인을 슬로건 → **가치제안**으로 교체(무엇을·누구에게). [[above-the-fold-priority]]
- 1차 CTA는 "14일 무료 체험" 하나만 버튼, 나머지는 텍스트 링크·앵커로 강등. [[hicks-law]]
- 일러스트를 **실제 제품 화면**으로 교체(비개발자 이해·신뢰).
- 폴드 근처에 신뢰 신호 한 줄(사용 팀 수·평점) 삽입 — F9의 후기를 여기로 일부 끌어올림.

### F5 — 대비 교정
- Before: 서브텍스트 #b3b3b3(추정 약 2:1). → After: 최소 #595959급 이상으로 낮춰 4.5:1 확보. 실측으로 검증. [[wcag-aa-checklist]]

### F4 — 메뉴 인터랙션
- Before: hover로만 열림, 클릭 무반응. → After: **클릭으로 열림**(모바일은 탭). hover는 보조로만. 모바일 햄버거 내에서도 하위 메뉴 접근 가능하게. [[thumb-zone-mobile]] · [[jakobs-law]]

### F6 — 모바일 하단 바
- Before: 아이콘 ~20px, 간격 좁음. → After: 터치 영역 최소 44×44pt로 확대(아이콘은 작아도 hit area를 패딩으로), 인접 간격 확보. [[touch-target-size]]

### F7 — 가입 폼
**After (텍스트 와이어프레임)**
```
이름
[            ]
회사 이메일
[            ]
비밀번호  (8자 이상, 영문+숫자)   ← 규칙을 미리 안내
[            ]
회사명
[            ]
  · 각 필드 위 레이블(플레이스홀더 대체 아님)
  · blur 시 인라인 검증, 에러는 해당 필드 바로 아래
  · 에러 카피: "비밀번호는 8자 이상이어야 합니다" (원인+해결책)
[14일 무료 체험 시작하기]   ← 버튼 라벨을 결과로
```
[[form-ux-best-practices]] · [[microcopy-guidelines]]

### F8 — 빈 상태
- 검색 무결과 시: 헤딩("‘OO’에 대한 결과가 없어요") + 동기부여(대안 제안) + CTA(전체 사례 보기/검색어 지우기). [[empty-state-onboarding]]

### F9·F10 — 신뢰·명확성
- 후기를 요금제 표 **바로 옆·위**로 이동(결정 지점). [[landing-page-review-checklist]]
- 기능 3개를 "그래서 담당자가 얻는 것"으로 서술("출근 체크 자동화로 월말 정산 시간 절반"). [[nielsen-10-heuristics]]

## 레퍼런스 근거

- 근거 우선순위대로 지식 노드를 1차 근거로 인용했다(위 각 Finding 참조).
- 실물 사례 참고: 유사 B2B SaaS 랜딩의 "제품 스크린샷 히어로 + 단일 CTA + 폴드 신뢰 신호" 관습은 Mobbin·SaaS 갤러리에서 다수 확인된다 — [[reference-galleries]](관습 확인용, 심미 심사는 usability 근거로 신중).
- 폼·터치 타깃 권장값 규범: Material/HIG의 44/48 기준 — [[design-systems-authority]].
- 시장 관습은 기본 미적용(시장 미확인). **대상 시장이 한국으로 확인되면** 가입 폼의 회사 이메일·사업자 인증 관습 등 한국 맥락을 추가 검토한다 — [[korean-market-ux-patterns]](조건부).

## 자가 재검 (R1~R7)

- R1 위치: 10개 Finding 모두 위치 명시 ✓
- R2 근거: 모두 휴리스틱/법칙/기준 노드 인용 ✓
- R3 심각도: 모두 0~4 부여 ✓
- R4 개선안: 모든 미스에 Before/After 또는 텍스트 와이어프레임 ✓
- R5 심미/사용성 구분: 일러스트→실물은 "취향"이 아니라 이해·신뢰 근거로 환원 ✓
- R6 검증 필요 표기: F5(대비비)·F6(px) 실측 필요 명시 ✓
- R7 우선순위: 폴드 우선·심각도순 정합 ✓
- R8 핸드오프 프롬프트: 리포트 말미에 복붙 블록 포함 ✓ (아래)

성공 기준 충족: S1(인테이크 확보) · S2(모든 Finding 위치+근거+심각도) · S3(Executive Summary+우선순위표) · S4(심각도 3+에 Before/After, 히어로·폼에 텍스트 와이어프레임) · S5(구현 코드 0) · S6("검증 필요" 표기 2건) · S7(메모리 후보) · S8(핸드오프 프롬프트).

## 핸드오프 프롬프트 (구현자에게 복붙 · S8)

아래 블록을 그대로 LLM/코딩 에이전트/디자인 툴에 붙여넣어 구현을 지시한다. 자문가는 구현하지 않는다.

```text
[GOAL] TeamFit 랜딩페이지의 "14일 무료 체험" 가입 전환을 높인다. 폴드 위에서 무엇을 하는
서비스인지 5초 안에 이해되고, 1차 CTA 하나로 결정이 모이게 한다.

[CONTEXT] 서비스=중소기업용 근태·연차 관리 SaaS. 타깃=5~50인 회사 인사 담당자(비개발자,
엑셀 익숙). 플랫폼=데스크톱 웹 + 모바일웹. 시장=인테이크 미확인 → 보편 원칙 적용(한국 관습 미적용).

[수정 지시 — 심각도순]
1. (S4) 히어로 헤드라인을 슬로건→가치제안으로 교체: "출퇴근·연차·근무표를 한곳에서 —
   5~50인 팀을 위한 근태·연차 관리 SaaS". 근거: 폴드 우선(조회 80%가 폴드 위).
2. 폴드 1차 CTA를 "14일 무료 체험" 하나만 버튼으로, 데모·요금제는 텍스트 링크/앵커로 강등.
   근거: Hick의 법칙(경쟁 CTA 축소).
3. 히어로 우측 일러스트를 실제 제품 화면(근무표/연차 승인 대시보드) 스크린샷으로 교체.
   근거: 랜딩 Above-fold "제품 비주얼" 요소.
4. 상단 메뉴 드롭다운을 클릭(모바일 탭)으로 열리게. hover 전용 금지. 근거: 터치 접근성/Jakob.
5. 히어로 서브텍스트 대비를 4.5:1 이상으로(현재 #b3b3b3 추정 약 2:1 → #595959급 이상).
   근거: WCAG 1.4.3. [실측 후 확정]
6. 모바일 하단 아이콘 터치 영역을 44x44pt 이상, 인접 간격 확보. 근거: WCAG 2.5.8/터치 타깃.
   [실측 후 확정]
7. 가입 폼: 필드 위 레이블(플레이스홀더 레이블 금지), 비밀번호 규칙 사전 안내, blur 인라인
   검증, 에러는 필드 바로 아래 "원인+해결책". 버튼 라벨 "시작하기"→"14일 무료 체험 시작하기".
8. 고객 후기를 요금제 표 옆/위(결정 지점)로 이동. 검색 무결과에 빈 상태(헤딩+대안+CTA) 추가.

[LAYOUT/COLOR] 히어로=좌 텍스트 / 우 제품 스크린샷 2단. 1차 CTA만 강조색, 2차는 텍스트 링크.
대비 하한 본문 4.5:1 · UI 컴포넌트 3:1.

[보존할 것 — 바꾸지 말 것]
- 로고 좌상단 배치, "로그인/무료 체험" 우상단 위치(관습에 맞음).
- 기능 3종의 구성 자체(출퇴근·연차·근무표) — 라벨을 혜택 문장으로 다듬되 항목은 유지.
- 요금제 표 구조.

[CONSTRAINTS] 접근성: 본문 대비 ≥4.5:1, 터치 타깃 ≥44pt, 키보드 포커스 표시. 톤: 확정 시
브랜드 가이드 따름(미정이면 현행 존댓말 유지). 시각 스타일 구현이 필요하면 MengTo/Skills
`landing-page` 구조 참고(코드 구현은 구현자 몫).

[완료 판정 기준]
- 폴드 위만 보고 "근태·연차 관리 SaaS"임을 5초 안에 회상 가능한가?
- 폴드 위 1차 버튼이 하나로 위계가 잡혔는가?
- 서브텍스트 대비 실측 ≥4.5:1, 모바일 아이콘 터치 영역 ≥44pt인가?
- 가입 폼 레이블이 필드 위에 있고 검증이 blur 시점·에러가 필드 아래인가?
```
