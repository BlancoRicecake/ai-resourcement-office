# 예시 입력 ② — 기존 화면 리뷰용 화면 묘사

(시나리오 B: 화면 리뷰. 스크린샷 대신 화면을 섹션별 텍스트로 상세 묘사한 것. 자문가는 이 묘사만으로 진단한다.)

## 리뷰 대상

- **서비스**: 팀핏(TeamFit) — 중소기업용 근태·연차 관리 SaaS.
- **타깃 사용자**: 5~50인 규모 회사의 인사 담당자(비개발자, 엑셀에 익숙). 가입 결정자는 대표 또는 총무.
- **핵심 목표(1차 전환)**: 랜딩 방문자가 "14일 무료 체험 시작"으로 가입하는 것.
- **플랫폼**: 데스크톱 웹 + 모바일웹(담당자가 외부에서 폰으로도 확인).
- **리뷰 대상 화면**: 마케팅 랜딩페이지(그리고 가입 폼).

## 화면 묘사 (위에서 아래로)

### 상단 바 (고정)
- 왼쪽에 로고 "TeamFit". 오른쪽에 메뉴 항목 5개: 기능, 요금제, 고객사례, 블로그, 회사소개.
- "기능"과 "요금제"에는 하위 메뉴가 있는데, **마우스를 올려야(hover) 드롭다운이 펼쳐진다.** 클릭해도 아무 일도 일어나지 않는다.
- 맨 오른쪽에 "로그인" 텍스트 링크와 파란 "무료 체험" 버튼.

### 히어로 (첫 화면, 폴드 위)
- 큰 글씨 헤드라인: **"일하는 방식의 새로운 기준."**
- 그 아래 작은 회색 글씨(연한 회색 #b3b3b3, 흰 배경): "TeamFit과 함께 더 나은 팀을 만드세요."
- 버튼이 **세 개 나란히**: 파란 "무료 체험 시작", 회색 테두리 "데모 영상 보기", 회색 테두리 "요금제 확인". 셋 다 비슷한 크기.
- 오른쪽에 추상적인 일러스트(사람들이 웃고 있는 그림). 제품 화면은 보이지 않는다.
- **이 화면 어디에도 "근태·연차 관리 SaaS"라는 설명이 없다.** 무엇을 하는 서비스인지 히어로만 봐선 알 수 없다.

### 스크롤 1~2번 아래
- "왜 TeamFit인가" 섹션. 여기서야 비로소 "출퇴근 기록, 연차 신청·승인, 근무표를 한곳에서" 라는 실제 기능 설명이 나온다.
- 기능 3개를 아이콘 + 짧은 제목으로 나열("간편한 출퇴근", "빠른 연차", "한눈에 근무표"). 각 기능이 구체적으로 뭘 해 주는지는 없다.

### 스크롤 3~4번 아래
- 요금제 표.
- 그 아래 고객 후기 2개("도입 후 편해졌어요 - OO물산 김OO"). **후기는 페이지 거의 맨 아래, 푸터 바로 위에 있다.**

### 가입 폼 (무료 체험 클릭 시 이동하는 페이지)
- 입력칸 4개: 이름, 회사 이메일, 비밀번호, 회사명. **각 입력칸에는 레이블이 없고, 회색 플레이스홀더 글씨로만("이름", "회사 이메일" …) 안내한다.** 입력을 시작하면 안내 글씨가 사라진다.
- 비밀번호 규칙(8자 이상 등) 안내가 화면에 없다. **제출 버튼을 눌러야만** "비밀번호가 너무 짧습니다" 같은 에러가 상단에 빨간 줄로 뜬다.
- 제출 버튼 "시작하기".

### 모바일웹에서의 모습
- 상단 바는 햄버거 메뉴로 바뀐다.
- 히어로의 버튼 3개가 세로로 쌓인다.
- 하단에 고정된 작은 아이콘 바(홈·검색·문의·계정)가 있는데, **아이콘이 작고(약 20px) 서로 바짝 붙어 있어** 옆 아이콘이 자주 눌린다.
- 고객사례 페이지에서 검색을 했는데 결과가 없을 때, **화면이 그냥 텅 비어** 아무 안내도 없다("검색 결과 없음"조차 없다).

## 요청

이 랜딩페이지와 가입 폼을 진단해 달라. 무료 체험 가입 전환을 높이는 게 목표다.
