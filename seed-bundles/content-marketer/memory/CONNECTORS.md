# CONNECTORS

발행 플랫폼 연결 상태와 발행 선호를 저장하는 파일이다. 세팅 방법은 `worker/skills/publish-connectors.md`를 따른다.

## 중요

- 이 파일에 API 키, 토큰, 비밀번호를 저장하지 않는다. 자격증명은 사용자의 로컬 환경변수·`.env`에만 둔다.
- 여기에는 "어떤 플랫폼을 연결했는지"와 "발행을 어떻게 처리할지"만 기록한다.

## 연결 상태

- 아직 연결된 플랫폼 없음. 기본은 발행 패키지(수동 게시).

기록 형식:

```md
| 플랫폼 | 연결 여부 | 자동 발행 가능성 | 비고 |
| --- | --- | --- | --- |
| WordPress | 미연결 | 현실적 | |
| 뉴스레터 | 미연결 | 현실적 | |
| 인스타/스레드/링크드인 | 미연결 | 제한적(준비까지) | |
```

## 발행 선호

- 게시 방식: 미정 (초안 스테이징 후 게시 직전 확인 / 예약)
- 발행 전 컨펌: 항상 필요 (고정 원칙, 변경 불가)

## 업데이트 규칙

연결 상태나 발행 선호가 바뀌면 `CONNECTORS.md 후보`로 제안한다. 발행 전 컨펌 원칙은 사용자가 요청해도 해제하지 않는다.
