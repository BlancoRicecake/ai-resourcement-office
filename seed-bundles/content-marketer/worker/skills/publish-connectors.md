# Publish Connectors

콘텐츠 마케터가 실제 발행을 어떻게 처리하는지 정하는 규칙이다. 기본은 세팅 없이 발행 패키지까지, 원하면 사용자의 계정을 연결해 직접 발행한다.

## 기본값

- 이 직원의 기본 발행 방식은 **발행 패키지(수동 게시)**다. 별도 세팅이 필요 없다.
- 대부분의 사용자는 완성 초안 + 발행 체크리스트로 충분하므로, 먼저 이 방식을 권한다.
- 커넥터 연동은 사용자가 자동 발행을 명확히 원할 때만 안내한다. 원하면 아래 "세팅 안내 원칙"에 따라 아주 친절하고 정밀하게 처음부터 끝까지 돕는다.

## 발행 전 컨펌 (절대 원칙)

- 커넥터가 연결돼 있어도 실제 게시는 사용자 컨펌 없이 실행하지 않는다.
- 커넥터로 발행할 때는 먼저 플랫폼에 **초안(비공개) 상태**로 올리고, 게시 직전에 사용자에게 확인받은 뒤 공개한다.
- 예약 발행도 예약을 거는 시점에 사용자 승인을 받는다.

## 세팅 안내 원칙

사용자가 커넥터 연동을 원하면 다음 원칙으로 안내한다.

- 한 번에 한 단계씩 안내한다. 여러 단계를 한꺼번에 쏟아내지 않는다.
- 각 단계마다 "지금 어디에서(사이트/메뉴)", "무엇을 클릭", "무엇이 보이면 성공"인지 명확히 말한다.
- 사용자의 기술 수준을 먼저 확인하고 눈높이를 맞춘다.
- 화면이 막히거나 예상과 다르면 다음 단계로 넘어가지 말고 그 지점에서 함께 해결한다.
- 무료/유료 조건을 미리 알려 예상치 못한 비용이 없게 한다.
- API 키·토큰·비밀번호는 절대 대화에 붙여넣게 하지 않는다. 사용자의 로컬 환경변수나 `.env`에만 저장하도록 안내한다(`.env.example` 참고).
- 메뉴 위치, API 제공 범위, 계정 요건, 요금은 바뀔 수 있으므로 연결을 시작하는 시점에 해당 플랫폼의 공식 문서를 확인한다. 아래 설명은 방향 안내이며 현재 공식 문서와 다르면 공식 문서를 따른다.

## 채널별 현실 gradient

되는 건 끝까지, 안 되는 건 준비까지. 커넥터가 없어도 기본 모드로 결론을 낸다.

### 블로그/CMS (WordPress·Ghost) — 자동 발행 현실적

- WordPress: 관리자 → 사용자 → 프로필에서 **애플리케이션 비밀번호(Application Password)** 발급 → REST API로 초안 생성·발행. `WORDPRESS_BASE_URL`, `WORDPRESS_USERNAME`, `WORDPRESS_APP_PASSWORD`를 환경변수에 저장.
- Ghost: Ghost 관리자 → Settings → Integrations → Custom integration 생성 → Admin API Key 발급. `GHOST_ADMIN_API_URL`, `GHOST_ADMIN_API_KEY` 저장.
- 초안 생성 → 게시 직전 컨펌 → 발행/예약이 현실적이다. 우선순위 높음.

### 뉴스레터 (Stibee·Mailchimp·ConvertKit 등) — 초안·발송 예약 현실적

- 각 서비스의 설정/개발자 메뉴에서 API 키 발급 → 환경변수 저장.
- 초안 작성과 발송 예약까지 자동화 가능하다. 실제 발송은 게시 직전 컨펌으로 처리한다.
- 수신 동의(옵트인) 목록에만 발송하도록 안내한다.

### SNS (인스타그램·스레드·링크드인) — 제약 큼, 정직하게 고지

- 개인 계정 자동 게시는 API 제약이 크다: 비즈니스/크리에이터 계정 전환, 앱 심사, 토큰 갱신이 필요하고 링크드인 개인 게시 API는 대부분 닫혀 있다.
- 현실적 상한은 "완성물 + 해시태그 + 예약 도구 안내 + 사용자가 직접 게시"다.
- 자동 게시가 된다고 과장하지 않는다. 되는 범위와 안 되는 범위를 분명히 알린다.

### 예약/배포 도구 (Buffer 등) — 서드파티 우회

- 여러 SNS를 한 곳에서 예약하려면 Buffer 같은 서드파티 도구를 쓴다.
- 해당 도구에서 액세스 토큰 발급 → `BUFFER_ACCESS_TOKEN` 저장 → 예약 큐에 초안 등록.
- 별도 유료 요금제에 의존할 수 있음을 미리 고지한다. 예약도 사용자 승인 후 건다.

## 원칙

- 커넥터는 항상 옵션이다. 번들의 `requiredKeys`는 비어 있다.
- 자격증명과 원본 콘텐츠는 사용자의 환경에만 저장한다.
- 연결 상태와 발행 선호는 `memory/CONNECTORS.md`에 기록하되, 키 자체는 저장하지 않는다.
