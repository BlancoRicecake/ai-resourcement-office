# Channel Format Rules

채널별 작성 규격이다. 초안 작성 단계에서 해당 채널 규격을 따른다. 모든 채널에서 미검증 수치·사례는 `[검증 필요]`로 표시하고, 컨펌 게이트 항목은 `claim-safety-rules.md`를 따른다.

## 블로그 / SEO 글

- 제목 후보 3개: 검색 의도형 1개, 문제 제기형 1개, 결과 약속형 1개
- 도입 후킹: 첫 2-3문장에서 독자의 문제를 짚는다
- 목차: H2/H3 구조로 논리 흐름을 만든다
- 본문: 각 섹션은 문제 → 근거 → 실행으로 전개한다
- FAQ: 독자가 실제로 검색할 질문 3-5개와 답
- 메타 설명: 150자 이내, 핵심 키워드와 가치 포함
- 본문 CTA: 글 흐름과 연결된 자연스러운 행동 유도 1개
- (참고: 이 블로그 규격은 제목 후보·목차·FAQ·메타 설명 구조를 표준으로 삼는다.)

## 인스타그램

- 첫 줄 후킹: 피드에서 멈추게 하는 한 문장
- 캐러셀 구성: 슬라이드별 한 메시지(표지 → 문제 → 근거/사례 → 해법 → CTA)
- 캡션: 후킹 반복 + 맥락 + 행동 유도
- 해시태그 후보: 대·중·소 규모 섞어 10-15개 제안(확정은 사용자)

## 스레드(Threads)

- 첫 게시물: 짧고 강한 단문 후킹
- 이어쓰기: 한 게시물 = 한 생각, 자연스러운 연결
- 마지막 게시물: 행동 유도 또는 대화 유도 질문

## 링크드인

- 도입: 전문가 관점의 한 문장 또는 짧은 경험
- 리듬: 짧은 문단과 줄바꿈으로 모바일 가독성 확보
- 핵심: 3문단 안에 인사이트를 전달한다
- 마무리: 단일 CTA 또는 의견을 묻는 질문
- 톤: 과장·자기자랑보다 구체적 관찰과 도움을 우선

## 뉴스레터

- 제목 후보 3개 + 프리헤더(미리보기 문구) 후보 2개
- 본문 구조: 인사 → 핵심 하나 → 근거/사례 → 단일 CTA
- 한 통에 한 목적. 여러 CTA로 분산하지 않는다
- 길이: 끝까지 읽을 수 있는 분량으로, 불필요한 반복을 줄인다
