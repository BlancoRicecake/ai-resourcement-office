# Tone Options

브랜드 보이스 기록(`memory/PROJECT.md`, `memory/USER.md`)이 없을 때 통합 제작안 안에서 톤 선택지를 제안한다. 기록이 있으면 그것을 우선하고 같은 질문을 반복하지 않는다.

## 제안 방식

- 이번 콘텐츠에 어울리는 톤 3-4개만 골라 제안한다(전부 나열하지 않는다).
- 각 톤마다: 느낌 / 적합한 채널 / 주의점을 한 줄씩 붙인다.
- 사용자가 톤을 지정하지 않았다면 추천안 하나와 대안 2-3개를 통합 제작안에 넣는다. 사용자가 이미 지정했다면 그대로 적용한다.
- 선택된 톤이 이후 반복 확인되면 `USER.md 후보`로 제안한다.

## 톤 선택지

| 톤 | 느낌 | 적합한 채널 | 주의점 |
| --- | --- | --- | --- |
| 전문가형 | 근거와 권위, 신뢰 | 블로그, 링크드인 | 어려워지지 않게, 실행 가능하게 |
| 친근한 조언형 | 옆에서 도와주는 말투 | 인스타, 뉴스레터 | 가벼워도 정보 밀도 유지 |
| 빠른 문제 해결형 | 바로 쓸 수 있는 해법 중심 | 블로그, 스레드 | 맥락 생략으로 오해 주지 않기 |
| 스토리텔링형 | 경험과 서사로 몰입 | 뉴스레터, 링크드인 | 서사가 길어져 핵심이 묻히지 않게 |
| 도발적 문제제기형 | 통념을 뒤집는 후킹 | 스레드, 인스타 | 자극만 남고 근거가 없지 않게 |

## 원칙

- 채널이 달라도 브랜드 보이스는 유지하고 형식만 바꾼다.
- 금지 표현(`memory/PROJECT.md`, `USER.md`)은 어떤 톤에서도 지킨다.
- 톤을 위해 사실을 왜곡하거나 과장하지 않는다.
