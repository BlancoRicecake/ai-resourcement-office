# Editorial Planning Rules

콘텐츠 마케터가 개별 콘텐츠를 쓰기 전에 채널 전체의 콘텐츠 결을 설계·관리하는 규칙이다. 현재 상태는 `memory/EDITORIAL.md`에 저장한다.

## 1. 콘텐츠 필러

필러는 이 채널이 반복해서 다루는 3-5개의 핵심 주제 축이다.

- 제품 가치와 타깃 독자의 문제에서 도출한다.
- 너무 넓으면(예: "마케팅") 좁히고, 너무 좁으면(예: "10월 3주차 프로모션") 상위 축으로 올린다.
- 각 필러마다: 이름 / 다루는 문제 / 타깃 독자 / 이 필러가 미는 행동(CTA 방향)을 정한다.
- 필러 후보는 확정하지 말고 통합 제작안과 `EDITORIAL.md 후보`에 함께 제안한다. 별도 컨펌 턴을 만들지 않는다.

## 2. 에디토리얼 캘린더

언제 어떤 필러의 콘텐츠를 어떤 채널에 낼지 계획한다.

- 발행 주기는 `memory/PROJECT.md`의 발행 주기 기록을 따른다. 없으면 사용자에게 확인한다.
- 필러가 한쪽으로 쏠리지 않게 분산한다.
- 큰 채널(블로그)에서 만든 콘텐츠를 작은 채널(SNS·뉴스레터)로 재가공하는 배포 흐름을 함께 계획한다.
- 캘린더는 확정 일정이 아니라 제안이다. 사용자가 조정할 수 있게 초안으로 제시한다.

## 3. 주제 파이프라인

다음에 만들 콘텐츠 후보 목록이다.

- 각 후보: 제목 가안 / 소속 필러 / 채널 / 우선순위 / 근거(왜 지금 이 주제인가)를 적는다.
- 근거는 `memory/RESEARCH.md`의 주제 반응·트렌드 기록과 연결한다. 근거가 약하면 `[검증 필요]`로 표시한다.
- 그로스 마케터의 작업 지시서가 오면 해당 콘텐츠를 파이프라인에 편입하고 우선순위를 재정렬한다.

## 4. 개별 콘텐츠 배치

콘텐츠 요청이나 지시서가 오면 제작 전에 먼저 배치한다.

- 어느 필러에 속하는가
- 캘린더 어디에 놓이는가
- 기존 발행 콘텐츠(`CONTENT-LOG.md`)와 중복되지 않는가
- 배포 흐름(원본 채널 → 재가공 채널)이 어떻게 되는가

결이 아직 없으면 최소한의 필러 후보를 통합 제작안에 포함한다. 사용자가 나머지 제작 조건을 이미 명확히 지정했다면 필러 확인만을 위한 별도 턴을 만들지 않고 초안과 함께 검토받을 수 있다.
