# Growth Marketer Interface

그로스 마케터와 주고받는 계약이다. 연동 모드에서 따른다.

## 수신: 작업 지시서

그로스 마케터의 성장 실험 설계서 중 `하위 실행 직원 작업 지시서`는 다음 표로 온다.

| 실행 직원 | 맡길 일 | 입력 자료 | 산출물 | 성공 기준 | 컨펌 |
| --- | --- | --- | --- | --- | --- |
| 콘텐츠 마케터 | 문제 해결형 콘텐츠 작성 | 리서치 브리프, 메시지 | 블로그/SNS 초안 | 클릭 또는 리드 전환 | 필요 |

함께 읽는 설계서 맥락:

- 실험명
- 가설
- 타깃
- 핵심 메시지
- 채널
- 1차 KPI
- 실험 기간
- 브랜드 톤과 금지 표현

## 수신 처리 규칙

- 지시 대상 행이 `콘텐츠 마케터`인 것만 실행한다. 다른 직원(퍼포먼스·CRM 등) 몫은 실행하지 않고 알린다.
- 채널·타깃·핵심 메시지 중 하나라도 없으면 초안을 쓰지 않고 먼저 질문한다.
- 나머지 항목이 없으면 `가정`으로 표시하고 진행한다.
- 지시서의 성공 기준을 콘텐츠의 CTA·측정 지표와 연결한다.

## 반환: 콘텐츠 실행 결과 보고

성과 데이터를 받으면 그로스 마케터가 실험 회고(`EXPERIMENTS.md`)에 바로 쓸 수 있게 다음 형식으로 돌려준다.

```md
# 콘텐츠 실행 결과 보고

- 연결 실험: (작업 지시서의 실험명, 없으면 "단독 작업")
- 발행 콘텐츠: | 채널 | 제목 | 발행일 | 링크 |
- 성과 지표: (사용자 제공 조회/클릭/전환 수치. 없으면 "데이터 필요", 추정 금지)
- 지시서 성공 기준 대비: 달성 / 미달 / 판단 불가(사유)
- 배운 점: (후킹, 주제, 채널 반응)
- 다음 콘텐츠 제안:
- 그로스 마케터 전달 사항: (EXPERIMENTS.md 후보 형식)
- 메모리 업데이트 후보: (CONTENT-LOG.md 등)
```

## 경계

- 성장 전략·실험 구조·예산 배분 판단은 그로스 마케터의 몫이다. 그 판단이 필요하면 되돌려 보낸다.
- 콘텐츠 결(필러·캘린더) 관리와 콘텐츠 제작·발행은 콘텐츠 마케터의 몫이다.
