# Research and Evidence Rules

콘텐츠를 쓰기 전에 필요한 리서치의 범위와 근거 상태를 정하는 규칙이다. 최신 사실이나 외부 주장을 사용하지 않는 순수 편집 작업이 아니라면 초안 전에 적용한다.

## 1. 리서치 필요도 판단

- **필수**: 최신 정책·가격·시장 수치·플랫폼 기능·트렌드·경쟁사 비교·전문가 인용·규제 산업 주장
- **권장**: SEO 글, 새로운 주제, 고객 문제를 일반화하는 글, 사용자가 근거 자료를 주지 않은 글
- **생략 가능**: 사용자가 제공한 확정 원문을 요약·재가공하거나 이미 승인된 브랜드 문구만 편집하는 작업

리서치를 생략할 때는 `리서치 생략 — 제공 자료만 사용`이라고 명시한다.

## 2. 초안 전 확인 항목

가능한 범위에서 다음을 확인한다.

1. 검색·질문 의도: 독자가 실제로 해결하려는 문제
2. 고객 언어: 인터뷰·문의·리뷰에서 반복되는 표현
3. 기존 콘텐츠 공백: 이미 흔한 설명과 새로 보탤 수 있는 관점
4. 핵심 주장 근거: 수치·사례·기능·인용을 뒷받침하는 출처
5. 최신성: 정책·제품·플랫폼 정보의 확인 날짜

검색 도구나 원문이 없으면 내용을 추정하지 않는다. 필요한 조사 질문과 사용자가 제공해야 할 자료를 짧게 요청한다.

## 3. 출처 우선순위

1. 공식 문서·공식 통계·원문 연구
2. 사용자가 제공한 내부 자료(외부 공개 가능 여부 별도 확인)
3. 신뢰할 수 있는 업계 보고서·전문 매체
4. 커뮤니티·SNS·경쟁 콘텐츠는 고객 언어와 아이디어 참고용

검색 결과 요약이나 출처 불명의 재인용을 핵심 근거로 사용하지 않는다. 외부 출처는 제목·발행 주체·URL·확인일을 기록한다.

## 4. 주장 검증표

초안 전에 주요 주장을 다음 표로 정리하고, 발행 전 다시 감사한다.

```md
| 주장 | 유형 | 근거/출처 | 상태 | 처리 |
| --- | --- | --- | --- | --- |
| 예시 주장 | 기능/수치/사례/의견 | 출처 또는 사용자 제공 자료 | 확인 / 검증 필요 / 제외 | 유지 / 완화 / 삭제 |
```

- 근거가 확인된 사실만 단정형으로 쓴다.
- 합리적 조언이나 해석은 사실과 구분해 `권장`, `가능성이 있다`, `이 글에서는`처럼 표현한다.
- 확인할 수 없는 수치·인용·고객 사례는 `[검증 필요]`로 표시하거나 제거한다.
- 내부 자료를 근거로 쓸 때는 공개 가능 여부를 사용자에게 확인한다.

## 5. SEO 콘텐츠 추가 확인

- 핵심 검색 의도와 보조 질문을 먼저 정의한다.
- 키워드 반복보다 질문에 완결되게 답하는 구조를 우선한다.
- 검색량·난이도 수치가 없으면 임의로 만들지 않고 `데이터 필요`로 표시한다.
- 경쟁 글의 문장·구조를 복제하지 않고, 정보 공백과 독자 행동을 기준으로 차별화한다.

## 6. 출력

기본 `콘텐츠 발행 패키지`에 다음을 포함한다.

- 리서치 요약 또는 리서치 생략 사유
- 사용한 출처 목록
- 주장 검증표
- 발행 전에 사용자가 확인해야 할 `[검증 필요]` 항목
