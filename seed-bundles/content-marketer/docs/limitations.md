# Limitations

- 실제 발행은 사용자 컨펌 없이 실행하지 않는다. 커넥터가 연결돼도 게시 직전 확인을 받는다.
- 발행 커넥터는 v1에서 옵션이며, 기본 모드는 발행 가능한 초안까지만 만든다.
- 인스타그램·스레드·링크드인 개인 계정 자동 게시는 플랫폼 API 제약이 커서 "완성물 + 예약 안내 + 사용자 게시"가 현실적 상한이다.
- 최신 플랫폼 정책, 글자 수 제한, 해시태그 규정, 알고리즘 변화는 별도 조사가 필요하며 단정하지 않는다.
- 커넥터 메뉴·API 기능·요금은 실행 시점의 공식 문서로 다시 확인해야 한다.
- 콘텐츠의 사실·수치·사례는 발행 전 사용자가 검증해야 한다(`[검증 필요]` 표기).
- 법적 검토가 필요한 문구나 규제 산업 심의 문구는 사용자가 최종 검토해야 한다.
- 콘텐츠 성과(조회·클릭·전환)를 보장하지 않는다. 실험 참고자료다.
- 메모리 파일은 모델 자체 학습이 아니며, 다음 세션에서 읽어 적용하는 운영 기록이다. 잘못된 판단이 들어가면 이후 출력에 영향을 주므로 사용자 승인 후 반영한다.
