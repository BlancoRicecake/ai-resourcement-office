# Build Log

## Source Problem

그로스 마케터는 성장 실험을 설계하고 `하위 실행 직원 작업 지시서`를 만들지만, 그 지시서를 받아 실제 콘텐츠를 만들고 발행할 실행 직원이 없었다. 콘텐츠 마케터는 그 첫 실행 직원이다.

기존 SEO/GEO 브리프 생성기는 브리프(제목 후보·목차·FAQ·메타 설명)까지만 만들었다. 콘텐츠 마케터는 그 브리프 구조를 계승해 발행 가능한 완성 초안과 발행까지 확장한다.

## MVP Scope

- 콘텐츠 결(필러·에디토리얼 캘린더·주제 파이프라인)을 소유·관리한다.
- 그로스 마케터의 작업 지시서 또는 단독 요청을 받아 블로그/SEO, 인스타그램, 스레드, 링크드인, 뉴스레터 초안을 만든다.
- 기본은 발행 패키지(수동 게시). 옵션으로 사용자가 CMS·뉴스레터·SNS·예약 도구를 연결하면 게시 직전 컨펌을 받아 발행한다.
- 성과를 그로스 마케터에게 보고 가능한 형식으로 돌려주고, 메모리 업데이트 후보를 제안한다.

## Non-Goals (v1)

- 플랫폼 자동 발행을 사용자 컨펌 없이 실행하지 않는다.
- 필수 API 키를 요구하지 않는다(`requiredKeys: []`). 커넥터는 항상 옵션.
- 인스타그램·스레드·링크드인 개인 계정 자동 게시를 과장하지 않는다(API 제약 고지).
