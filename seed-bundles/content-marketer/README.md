# 콘텐츠 마케터 번들

처음 사용하는 분은 [처음사용하기.md](./처음사용하기.md)부터 읽으세요.

콘텐츠 마케터는 광고비 없이 고객을 데려오는 콘텐츠를 기획하고, 채널별로 바로 발행할 수 있는 초안을 만들고, (사용자가 연결하면) 직접 발행하고, 성과를 회고하는 AI 직원이다.

아이디어만 나열하는 작가가 아니라, 콘텐츠 결(필러·캘린더)을 소유하고 기획 → 제작 → 발행 → 회고를 스스로 도는 실행 직원이다. 성장 전략 설계는 `그로스 마케터`의 일이고, 콘텐츠 마케터는 그 지시서를 받아 실행하거나 단독으로 콘텐츠를 운영한다.

## 사용 목적

- 블로그/SEO 글, 인스타그램·스레드·링크드인 게시물, 뉴스레터를 발행 가능한 완성 초안으로 만든다.
- 검색 의도·고객 언어·경쟁 콘텐츠 공백을 조사하고, 핵심 주장을 출처와 연결한 뒤 초안을 쓴다.
- 채널 전체의 콘텐츠 필러, 에디토리얼 캘린더, 주제 파이프라인을 관리한다.
- 그로스 마케터의 작업 지시서를 받아 콘텐츠를 만들고, 결과를 실험 회고에 쓸 수 있는 형식으로 돌려준다.
- 기본은 발행 패키지(사용자 수동 게시), 원하면 사용자의 CMS·뉴스레터 계정을 연결해 게시 직전 컨펌을 받고 직접 발행한다.
- 브랜드 보이스, 금지 표현, 반응이 좋았던 패턴을 메모리 업데이트 후보로 정리한다.

## 두 가지 사용 모드

- 연동 모드: 그로스 마케터의 성장 실험 설계서 중 `하위 실행 직원 작업 지시서`를 입력으로 받는다.
- 단독 모드: 지시서 없이 사용자 요청만으로 시작한다. 목표·타깃·채널·핵심 메시지·CTA를 담은 간이 브리프를 먼저 컨펌받은 뒤 초안을 쓴다.

## 포함 파일

- `AGENTS.md`: 폴더 연결 진입점 (런타임이 폴더를 열 때 자동으로 읽는 지시문 로드 순서 안내)
- `worker/agent.md`: Claude, Codex 등에 넣어 사용할 수 있는 직원 지시문
- `worker/agent.json`: 직원 메타데이터
- `worker/skills/`: 콘텐츠 기획, 리서치·근거 검증, 채널 규격, 톤, 수정, 주장 안전, 발행 커넥터, 그로스 마케터 연동 규칙
- `memory/`: 사용자 선호, 프로젝트 기준, 콘텐츠 결/캘린더, 발행 이력, 연결 상태, 결정, 리서치와 선택 로드형 지식 노드
- `examples/input/`: 예시 요청과 예시 작업 지시서
- `examples/output/`: 예시 산출물
- `docs/`: 비용, 보안, 한계, 제작 기록, 수동 평가 케이스

## 사용 방법

1. `worker/agent.md` 내용을 Claude 프로젝트 지침, Codex 지침, 또는 사용하는 에이전트 런타임의 시스템 지침에 넣는다.
2. `memory/` 파일에 기존 프로젝트 정보가 있으면 함께 제공한다.
3. 제품, 타깃 독자, 목표 행동, 사용할 채널, 브랜드 톤을 제공한다. 그로스 마케터의 작업 지시서가 있으면 함께 넣는다.
4. 콘텐츠 마케터가 한 번에 제안한 통합 제작안(브리프·필러·채널·톤·CTA)을 확인하고, 콘텐츠 발행 패키지와 메모리 업데이트 후보를 검토한다.
5. 사용자가 승인한 콘텐츠만 실제 발행하고, 승인한 메모리만 파일에 반영한다.

## 예시 입력

```txt
고객 인터뷰 기록 SaaS의 무료 체험 유입을 위해, 블로그 글 1편과 링크드인 배포문이 필요해. 타깃은 초기 스타트업 PM이고, 톤은 전문가지만 친근했으면 좋겠어.
```

## 주의

이 번들은 호스팅 서비스가 아니다. 사용자는 자신의 Claude, Codex, OpenAI API, 또는 기타 모델 환경에서 직접 실행한다. SNS·블로그·뉴스레터 발행, 계정 연결, 발행 콘텐츠의 사실 검증, 법적 표현 검토, 개인정보 처리는 사용자가 최종 책임진다. 커넥터를 연결해도 실제 게시는 사용자 컨펌 후에만 이루어진다.
