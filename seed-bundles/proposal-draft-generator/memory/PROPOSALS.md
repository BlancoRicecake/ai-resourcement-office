# PROPOSALS

과거 제안서 이력과 결과를 기록하는 파일이다. 수주/탈락 결과와 원인을 분석하고, 효과적이었던 구조·논리·문구를 다음 제안서에 재사용한다.

## 기록 원칙

- 제안서를 전달한 뒤 결과(수주, 탈락, 보류)가 확인되면 원인 분석과 함께 기록한다.
- 효과적이었던 목차 구조, 설득 논리, 문구는 재사용 조건과 함께 남긴다.
- 탈락 원인은 추측이면 `가정`으로, 고객 피드백이 있으면 `근거`로 표시한다.

## 기록 형식

```md
## YYYY-MM-DD 제안 건명

- 고객 유형:
- 고객 문제:
- 제안 범위(포함/제외):
- 가격/일정 초안:
- 결과: 수주 / 탈락 / 보류
- 원인 분석(가정/근거 표시):
- 효과적이었던 구조·논리·문구:
- 효과 없던 부분:
- 배운 점:
- 다음 제안 반영:
```

## 제안 기록

- 아직 기록된 제안 없음.
