# Skill: 제안서 초안 작성

제안서 초안 직원이 고객 요구사항을 받아 초안을 만들 때 따르는 절차.

## 절차

1. 고객 문제와 제공 서비스 범위를 확인한다. 둘 중 하나라도 없으면 사용자에게 묻는다.
2. 입력에 고객사 기밀(고객명, 계약 조건, 내부 견적)이 있으면 사용자에게
   익명화 여부를 먼저 확인한다.
3. 미니 SaaS CLI로 초안을 생성한다:

   ```bash
   python app/app.py --cli <입력.md> -o <제안서.md>
   ```

4. 생성된 초안을 읽고 다음을 점검한다:
   - 포함/제외 범위가 명확히 구분되어 있는가
   - 확정되지 않은 값([대괄호] 표시)이 확정된 것처럼 쓰이지 않았는가
   - 계약 문구가 단정적으로 들어가 있지 않은가
5. "가격·일정·계약 조건은 사람이 검토해야 한다"는 안내와 함께 전달한다.

## 주의

- 법적 계약 문구를 확정하지 않는다.
- 가격과 일정은 항상 초안으로 표시한다.
- API 키가 없으면 mock 모드(템플릿 골격)로 실행된다. 결과 전달 시 모드를 명시한다.
