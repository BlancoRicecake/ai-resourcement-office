# Handoff: 기획자/작가/제작자에게 전달할 성과 피드백

성과 분석 결과를 각 직원이 바로 실행할 수 있는 지시로 나눠 전달한다.
"더 잘 만들자" 같은 추상적 지시는 금지하며, 모든 지시에 근거와 검증 지표를 붙인다.

## 공통 헤더

- 분석 대상 영상:
- 비교 기준(채널 중앙값/기간):
- 핵심 발견 요약 (1-3개):
- 데이터 출처(수동 입력/CSV/API):

## 컨텐츠 기획자에게

- 지시: (예: 다음 기획에서 OO 유형 주제를 우선 후보에 올릴 것)
- 근거: (어떤 지표가 채널 평균 대비 얼마나 달랐는지)
- 검증 지표: (다음 영상에서 무엇을 보면 이 판단이 맞았는지 알 수 있는지)

## 영상 원고 작가에게

- 지시: (예: 첫 15초 후킹을 OO 방식으로 바꿀 것, N분대 구조 유지)
- 근거: (유지율/이탈 구간 데이터)
- 검증 지표: (첫 30초 유지율, 주요 이탈 구간 변화)

## 영상 제작자에게

- 지시: (예: 썸네일 문구를 OO 방향으로, 컷 리듬 조정)
- 근거: (CTR/노출 데이터, 이탈 구간)
- 검증 지표: (CTR, 해당 구간 유지율)

## 우선순위 규칙

- 이번 사이클에 반영할 지시는 팀 전체 합산 1-3개로 제한한다.
- 나머지 발견은 `[대기]`로 표시해 다음 사이클 후보로 남긴다.
- 사용자가 확정한 채널 규칙(feedback-rules 저장분)과 충돌하는 지시는
  전달하기 전에 사용자에게 먼저 확인받는다.
