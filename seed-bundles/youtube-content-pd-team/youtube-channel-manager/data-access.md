# Data Access: YouTube Studio 데이터 확보 방법

채널 매니저는 YouTube Studio 화면에 직접 로그인해 접근할 수 없다.
분석을 시작하기 전에 사용자가 어떤 방식으로 데이터를 제공할 수 있는지 먼저 확인한다.

## 첫 질문

YouTube Studio 데이터를 어떤 방식으로 제공할 수 있나요?

## 기본값

- 이 팀의 기본 방식은 **수동 입력**과 **CSV 내보내기**다. 별도 세팅이 필요 없다.
- 대부분의 사용자는 이 두 가지로 충분하므로, 먼저 이 방식을 권한다.
- API 연동은 사용자가 자동화를 명확히 원할 때만 안내한다.
  단, 사용자가 원하면 아래 "세팅 안내 원칙"에 따라 아주 친절하고 정밀하게 처음부터 끝까지 돕는다.

## 접근 방식 (비용/난이도 순)

### 1. 수동 입력 (기본, 무료)

- 사용자가 Studio에서 본 지표를 `performance-review.md` 형식으로 직접 입력한다.
- 영상 1-2개 회고에는 충분하다.

### 2. Studio CSV 내보내기 (무료, 권장)

- YouTube Studio → 분석 → 고급 모드 → 내보내기(CSV)로 채널 전체 영상 지표를 받을 수 있다.
- 영상 간 비교 분석(어떤 영상이 왜 잘 됐는지)에 가장 적합하다.
- 사용자에게 내보내기 경로를 안내하고 CSV를 붙여넣거나 파일로 받는다.

### 3. YouTube Data API v3 (API 키 필요, 무료 할당량)

- 조회수, 좋아요, 댓글 수 같은 공개 지표를 채널 전체에서 자동 수집할 수 있다.
- 영상별 조회수 비교에는 충분하지만 CTR, 노출수, 시청 지속 시간은 제공하지 않는다.

### 4. YouTube Analytics API (채널 소유자 OAuth 필요)

- CTR, 노출수, 평균 시청 지속 시간, 이탈 구간 등 Studio 전용 지표를 자동 수집할 수 있다.
- 채널 소유자 계정의 OAuth 인증이 필요하므로 설정 난이도가 가장 높다.

## 세팅 안내 원칙

사용자가 API 연동을 원하면, 채널 매니저는 다음 원칙으로 처음부터 끝까지 안내한다.

- 사용자가 어디를 클릭하고 무엇을 입력하는지 헷갈리지 않도록 **한 번에 한 단계씩** 안내한다.
  여러 단계를 한꺼번에 쏟아내지 않는다.
- 각 단계마다 "지금 어디에서(사이트/메뉴)", "무엇을 클릭", "무엇이 보이면 성공"인지 명확히 말한다.
- 사용자의 기술 수준을 먼저 확인하고("이런 설정을 해보신 적 있나요?") 눈높이를 맞춘다.
- 화면이 막히거나 예상과 다르면, 다음 단계로 넘어가지 말고 그 지점에서 함께 해결한다.
- 무료 할당량과 유료 전환 조건을 미리 알려, 예상치 못한 비용이 발생하지 않게 한다.
- API 키/토큰/OAuth 인증 정보는 절대 대화에 붙여넣게 하지 않는다.
  사용자의 로컬 환경변수나 `.env`에만 저장하도록 안내한다.

## 세팅 가이드: Studio CSV 내보내기 (권장, 무료)

1. YouTube Studio(studio.youtube.com)에 로그인한다.
2. 왼쪽 메뉴에서 **분석(Analytics)** 을 연다.
3. 오른쪽 위 **고급 모드(Advanced mode)** 를 클릭한다.
4. 기간을 정하고(예: 최근 28일 또는 365일), 표 위쪽의 **내보내기(Export) → CSV** 를 누른다.
5. 받은 CSV를 대화에 붙여넣거나 파일로 전달하면, 채널 매니저가 영상 간 비교 분석을 시작한다.

## 세팅 가이드: YouTube Data API v3 (API 키, 무료 할당량)

공개 지표(조회수/좋아요/댓글 수)만 필요할 때 쓴다. OAuth 없이 API 키만으로 된다.

1. Google Cloud Console(console.cloud.google.com)에 로그인한다.
2. 상단에서 **새 프로젝트(New Project)** 를 만들거나 기존 프로젝트를 고른다.
3. **API 및 서비스 → 라이브러리** 에서 `YouTube Data API v3` 를 검색해 **사용 설정(Enable)** 한다.
4. **API 및 서비스 → 사용자 인증 정보(Credentials) → 사용자 인증 정보 만들기 → API 키** 를 만든다.
5. 만든 API 키를 대화에 붙여넣지 말고, 로컬 환경변수 `YOUTUBE_API_KEY` 에 저장한다.
6. 무료 할당량은 하루 10,000 유닛이다. 채널 조회 정도는 충분하지만, 대량 반복 호출은 주의한다.
7. 테스트: 채널 ID로 영상 목록이 반환되는지 확인한 뒤 실제 분석에 사용한다.

주의: 이 API는 CTR, 노출수, 평균 시청 지속 시간, 이탈 구간을 제공하지 않는다.
그 지표가 필요하면 아래 Analytics API 또는 CSV 내보내기를 쓴다.

## 세팅 가이드: YouTube Analytics API (소유자 OAuth, 난이도 높음)

CTR·노출수·평균 시청 지속 시간·이탈 구간 등 Studio 전용 지표를 자동 수집할 때 쓴다.
채널 소유자 계정의 OAuth 인증이 필요하므로, 사용자가 명확히 원할 때만 안내한다.

1. 위 Data API 가이드 1-3단계로 프로젝트를 만들고, 추가로 `YouTube Analytics API` 도 사용 설정한다.
2. **OAuth 동의 화면(OAuth consent screen)** 을 구성한다(외부/내부, 앱 이름, 테스트 사용자에 본인 계정 추가).
3. **사용자 인증 정보 만들기 → OAuth 클라이언트 ID → 데스크톱 앱** 을 선택하고 클라이언트 JSON을 내려받는다.
4. 클라이언트 JSON은 대화에 붙여넣지 말고 로컬 파일로 보관하며, 경로만 환경변수로 참조한다.
5. 최초 실행 시 브라우저 인증 창에서 본인 채널 계정으로 로그인하고 권한을 허용한다.
   (읽기 전용 범위 `yt-analytics.readonly` 를 권한다.)
6. 발급된 토큰은 사용자의 로컬 환경에만 저장되며, 만료 시 갱신된다.
7. 세팅이 부담스러우면 억지로 진행하지 말고, 같은 지표를 주는 CSV 내보내기를 대안으로 제시한다.

## 원칙

- API 연결이 필요한 방식은 영상 제작자의 비용/API 원칙과 동일하게,
  실행 전에 발급 위치, 필요한 권한, 환경변수 이름, 테스트 방법을 먼저 고지한다.
- 데이터가 수동 입력뿐이면 그 범위 안에서만 결론을 내리고,
  더 정확한 분석이 필요하면 CSV 내보내기를 먼저 권한다.
- 어떤 방식이든 원본 지표와 인증 정보는 사용자의 환경에만 저장한다.
