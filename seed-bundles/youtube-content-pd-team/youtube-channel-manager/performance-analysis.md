# Performance Analysis: 상위 성과 영상 분석

단일 영상 회고를 넘어, 어떤 영상이 채널 평균 대비 높은 성과를 냈는지와
그 이유를 근거 있게 판단하는 절차다.

## 1. 상위 성과 영상 식별

- 채널 최근 영상(가능하면 10-20개)의 조회수를 모은다.
- 절대 조회수가 아니라 채널 중앙값 대비 배율로 비교한다.
  (예: 중앙값 1,000회인 채널에서 4,000회 영상 = 4x)
- 게시 후 경과 시간이 다르면 같은 기간 기준(예: 첫 7일 조회수)으로 보정한다.
- 노출수와 CTR 데이터가 있으면 "노출을 많이 받은 것"과 "클릭을 잘 받은 것"을 구분한다.

## 2. 차별점 비교

상위 성과 영상을 채널의 다른 영상들과 다음 축에서 비교한다.

- 주제: 다룬 소재, 시청자 문제, 검색/추천 수요
- 제목: 구조, 길이, 숫자/키워드 사용, 약속의 강도
- 썸네일: 문구, 구도, 색, 얼굴/사물 여부
- 후킹: 첫 30초 유지율, 도입 방식
- 구조: 섹션 수, 전개 리듬, 영상 길이
- 외부 요인: 게시 시점, 트렌드, 알고리즘 노출 변화

각 축에서 "상위 영상만 다르게 한 것"과 "다른 영상과 같았던 것"을 나눈다.

## 3. 근거 기준

- 표본이 적으면(비교 영상 5개 미만) 결론을 단정하지 않고 가설로 표시한다.
- 성과를 단일 원인으로 귀속하지 않는다. 차별점이 여러 개면 모두 나열하고
  지표로 뒷받침되는 순서로 우선순위를 매긴다.
  (예: CTR이 높았으면 제목/썸네일 가설이 우선, 유지율이 높았으면 후킹/구조 가설이 우선)
- 각 판단에는 "어떤 지표가", "채널 평균 대비 얼마나" 달랐는지를 함께 적는다.
- 지표로 확인할 수 없는 추측은 `[가설]`로 표시하고 다음 영상에서 실험으로 검증하게 한다.

## 4. 판단 → 행동 변환

분석 결과는 감상이 아니라 다음 영상에서 실행 가능한 지시로 바꾼다.

- 각 판단마다: 무엇을 바꿀지 / 누가 반영할지(기획자·작가·제작자) /
  어떤 지표로 검증할지 를 정한다.
- 한 번에 너무 많은 변수를 바꾸면 검증이 불가능하므로,
  다음 영상에 적용할 핵심 변경은 1-3개로 제한한다.
- 변환 결과는 `handoff-to-team.md` 형식으로 출력한다.
