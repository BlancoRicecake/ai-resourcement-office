# 유튜브 콘텐츠 PD팀

유튜브 콘텐츠를 기획부터 성과 회고까지 운영하기 위한 AI 직원 팀입니다.

## 구성원

- `youtube-content-planner/` - 컨텐츠 기획자
- `youtube-script-writer/` - 영상 원고 작가
- `youtube-video-producer/` - 영상 제작자
- `youtube-channel-manager/` - 채널 매니저

## 사용 흐름

1. 컨텐츠 기획자가 기존 채널 여부를 확인하고 채널 정체성, 분야, 시청자, 경쟁 환경을 파악합니다.
2. 최근 데이터와 신뢰 가능한 출처를 바탕으로 기획 근거를 수집합니다.
3. 사용자에게 핵심 선택 질문을 던져 기획 방향을 확정합니다.
4. 컨텐츠 기획자가 원고 작가에게 넘길 기획 브리프를 정리합니다.
5. 영상 원고 작가가 숏폼/롱폼과 원고 분위기를 확인한 뒤 초안을 작성합니다.
6. 사용자가 원고를 컨펌하거나 피드백하면 원고 작가가 정확한 위치에 반영합니다.
7. 컨펌된 원고를 영상 제작자에게 넘깁니다.
8. 영상 제작자가 제작 형식, 비용/API 필요 여부, 스타일 세팅을 확인합니다.
9. 사용자가 제작 초안을 컨펌하거나 피드백하면 영상 제작자가 반영합니다.
10. 컨펌된 제작 정보를 채널 매니저에게 넘깁니다.
11. 채널 매니저가 업로드 메타데이터와 YouTube Studio 지표 기반 개선안을 만듭니다.
12. 채널 매니저가 채널 평균 대비 상위 성과 영상을 분석해 차별점을 찾고,
    핵심 개선 지시 1-3개를 기획자, 원고 작가, 영상 제작자에게 전달합니다.
13. 각 직원은 사용자 피드백과 채널 매니저 피드백을 함께 반영하며
    다음 사이클을 시작합니다. 충돌 시 사용자 확정 규칙이 우선합니다.

## 입력 자료

- 채널 설명과 목표
- 기존 채널 URL 또는 신규 채널 아이디어
- 기존 원고, 썸네일, 업로드 제목, 설명란, 성과 메모
- 타깃 시청자
- 제품, 서비스, 브랜드 맥락
- 영상 주제 또는 레퍼런스
- 숏폼/롱폼 여부
- 원하는 원고 분위기
- 원고 피드백과 컨펌 여부
- 원하는 제작 형식
- 외부 API 사용 가능 여부와 예산 기준
- 선호하는 영상 스타일
- 제작 피드백과 컨펌 여부
- 업로드 후 CTR, 평균 시청 지속 시간, 이탈 구간 메모

## 책임 범위

이 팀은 사용자의 환경에서 실행되는 문서 기반 AI 직원 팀입니다. YouTube Studio에
직접 로그인해 접근하지 않으며, 기본은 사용자가 제공한 지표와 자료(수동 입력 또는
Studio 고급 모드 CSV 내보내기)를 기준으로 기획안, 원고, 제작 가이드, 운영 리포트를
만듭니다. 사용자가 원하면 YouTube Data API(공개 지표) 또는 YouTube Analytics
API(소유자 OAuth 필요)를 자신의 환경에 연결해 지표 수집을 자동화할 수 있으며,
연결 방법과 조건은 `youtube-channel-manager/data-access.md`를 따릅니다.
