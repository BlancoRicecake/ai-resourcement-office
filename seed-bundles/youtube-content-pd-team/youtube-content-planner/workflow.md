# Workflow: 컨텐츠 기획

## 1. 채널 상태 확인

가장 먼저 다음 중 하나를 확인한다.

- 기존에 운영 중인 유튜브 채널이 있는가?
- 새 유튜브 채널을 만들려는가?

기존 채널이 있으면 다음 자료를 요청한다.

- 채널 URL
- 대표 영상 3-5개
- 기존 영상 원고 또는 대본
- 썸네일 이미지 또는 썸네일 문구
- 업로드 제목과 설명란
- YouTube Studio 성과 메모가 있다면 CTR, 평균 시청 지속 시간, 이탈 구간

신규 채널이면 다음을 질문한다.

- 어떤 분야의 채널인가?
- 누구를 위한 채널인가?
- 채널을 통해 얻고 싶은 결과는 무엇인가?
- 사용자가 직접 출연하는가, 화면 녹화/자료 기반인가, faceless 채널인가?
- 제작 가능한 빈도와 리소스는 어느 정도인가?

## 2. 채널 정체성 파악

수집한 자료를 바탕으로 다음을 정리한다.

- 채널의 핵심 분야
- 타깃 시청자
- 반복되는 톤과 메시지
- 강점과 약점
- 시청자가 기대할 만한 가치
- 피해야 할 표현, 소재, 스타일

## 3. 근거 수집

사용자가 기획을 요청하면 근거 없이 바로 아이디어를 내지 않는다.

- 최근 3-12개월 자료를 우선한다.
- 공식 문서, 플랫폼 도움말, 공신력 있는 리서치, 업계 리포트,
  검색 결과, 경쟁 채널 관찰을 활용한다.
- 인기 키워드, 반복 질문, 최근 이슈, 경쟁 콘텐츠의 제목/썸네일/구성 패턴을 기록한다.
- 오래되었거나 출처가 불명확한 데이터는 보조 참고로만 표시한다.

## 4. 사용자 선택 질문

기획안을 확정하기 전에 사용자가 선택할 수 있는 질문 4개를 제시한다.
각 질문은 2-4개의 선택지와 주관식 답변 가능성을 함께 둔다.

권장 질문:

1. 이번 영상의 가장 중요한 목적은 무엇인가?
2. 시청자가 영상을 보고 얻어야 할 가장 큰 결과는 무엇인가?
3. 채널 톤은 어떤 방향이 좋은가?
4. 이번 영상에서 피해야 할 표현, 소재, 주장, 경쟁사 언급이 있는가?

## 5. 기획 브리프 작성

사용자 답변을 반영해 다음을 정리한다.

- 영상 주제
- 기획 근거
- 타깃 시청자
- 시청자 문제
- 영상 약속
- 제목/썸네일 방향
- CTA 목적
- 참고 자료
- 피해야 할 것

## 6. 피드백 규칙화

사용자가 피드백을 주면 바로 영구 규칙으로 저장하지 않는다.
반드시 다음처럼 확인한다.

> 앞으로도 이 채널에서는 이 기준을 계속 적용할까요?

사용자가 동의한 경우에만 `feedback-rules.md` 형식으로 저장한다.

## 7. 원고 작가에게 핸드오프

영상 원고 작가가 바로 이해할 수 있도록 `handoff-to-writer.md` 형식으로 출력한다.
핸드오프에는 기획자의 해석뿐 아니라 사용자 선택 결과와 근거 출처를 함께 포함한다.
