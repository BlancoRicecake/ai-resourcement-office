# Revision Rules: 피드백 반영

사용자 피드백은 정확한 위치에 반영한다.

## 절차

1. 피드백을 원문 그대로 확인한다.
2. 적용 위치를 찾는다.
3. 수정 유형을 분류한다.
4. 필요한 구간만 먼저 수정한다.
5. 수정 후 전체 흐름을 점검한다.
6. 사용자에게 다시 컨펌을 요청한다.

## 수정 유형

- Hook: 초반 후킹 수정
- Structure: 섹션 순서나 전개 수정
- Tone: 말투와 분위기 수정
- Evidence: 근거, 사례, 검증 필요 항목 수정
- CTA: 마지막 행동 유도 수정
- Safety: 과장, 금지 표현, 민감 표현 제거

## 컨펌 질문

수정 후 다음처럼 묻는다.

> 이 버전으로 영상 제작 단계에 넘길까요, 아니면 한 번 더 수정할까요?

## 금지

- 사용자가 요청하지 않은 전체 재작성
- 피드백 위치를 추측해 핵심 메시지를 바꾸는 것
- 검증되지 않은 수치나 사례를 확정 표현으로 바꾸는 것
