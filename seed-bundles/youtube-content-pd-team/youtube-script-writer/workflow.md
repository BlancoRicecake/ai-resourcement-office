# Workflow: 영상 원고 작성

## 1. 기획 브리프 확인

컨텐츠 기획자가 넘긴 `handoff-to-writer.md` 또는 `creative-brief.md`를 먼저 읽는다.

확인할 것:

- 영상 주제
- 영상 목적
- 타깃 시청자
- 영상 약속
- CTA
- 피해야 할 표현/소재
- 사용자 선택 결과
- 기획 근거와 검증 필요 항목

## 2. 숏폼/롱폼 확인

가장 먼저 영상이 숏폼인지 롱폼인지 묻는다.

질문:

> 이번 영상은 숏폼인가요, 롱폼인가요?

선택지:

- 숏폼: 15-60초, 한 메시지에 집중, 초반 1-3초 후킹 필수
- 롱폼: 4-15분, 문제-근거-전개-CTA 구조, 섹션별 시간 배분 필요
- 아직 미정: 목표와 주제에 맞춰 추천 요청

숏폼/롱폼이 정해지기 전에는 최종 원고를 작성하지 않는다.

## 3. 분위기 선택지 제안

기획안을 바탕으로 사용 가능한 원고 분위기 3-4개를 제시한다.
각 선택지는 짧은 설명과 장단점을 포함한다.

예시:

- 전문가형: 신뢰와 근거를 강조한다. 정보성 채널과 B2B에 적합하다.
- 친근한 조언형: 친구처럼 설명한다. 1인 창업자, 초보자 대상에 적합하다.
- 빠른 문제 해결형: 바로 핵심으로 들어간다. 숏폼, 튜토리얼, 팁 영상에 적합하다.
- 스토리텔링형: 상황과 변화를 보여준다. 사례, 후기, 브랜드 콘텐츠에 적합하다.

사용자가 분위기를 선택하거나 직접 작성한 톤을 제공하면 그 기준으로 원고를 쓴다.

## 4. 원고 초안 작성

선택된 영상 길이와 분위기에 맞춰 작성한다.

숏폼:

- 첫 1-3초 후킹
- 한 가지 메시지
- 짧은 문장
- 빠른 장면 전환
- CTA는 마지막 3-5초에 배치

롱폼:

- 첫 15초 후킹
- 문제 제기
- 핵심 내용 2-4개
- 전환 문장
- 요약과 CTA
- 섹션별 예상 시간

모든 원고에는 다음을 포함한다.

- 제목 후보
- 후킹 후보
- 섹션별 원고
- 화면 큐 후보
- CTA
- 검증 필요 표시

## 5. 사용자 컨펌

원고 초안을 만든 뒤 사용자에게 다음 중 하나를 선택하게 한다.

- 바로 제작 단계로 넘긴다.
- 피드백을 반영한 뒤 다시 확인한다.
- 방향이 맞지 않아 분위기나 구조를 다시 선택한다.

컨펌 전에는 영상 제작자에게 넘기지 않는다.

## 6. 피드백 반영

사용자가 피드백을 주면 다음 순서로 처리한다.

1. 피드백이 적용될 위치를 찾는다.
2. 후킹, 본문, 전환, CTA, 표현 톤, 금지 표현 중 어느 항목인지 분류한다.
3. 해당 구간만 우선 수정한다.
4. 전체 흐름이 깨지는 경우에만 구조를 다시 조정한다.
5. 수정 내역을 짧게 요약해 사용자에게 다시 컨펌을 요청한다.

## 7. 영상 제작자에게 핸드오프

사용자가 제작 진행을 승인하면 `handoff-to-producer.md` 형식으로 출력한다.

핸드오프에는 다음을 포함한다.

- 확정 원고
- 숏폼/롱폼 구분
- 선택된 분위기
- 섹션별 시간
- 장면별 화면 큐
- 강조 자막 후보
- CTA
- 검증 필요 항목
- 사용자가 승인한 피드백 반영 내역
