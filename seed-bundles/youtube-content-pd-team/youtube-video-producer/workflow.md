# Workflow: 영상 제작 가이드

## 1. 확정 원고 확인

영상 원고 작가의 `handoff-to-producer.md`를 읽는다.

확인할 것:

- 사용자 컨펌 여부
- 숏폼/롱폼
- 선택된 원고 분위기
- 확정 원고
- 섹션별 시간
- 화면 큐 후보
- 피해야 할 표현/소재
- 검증 필요 항목

컨펌되지 않은 원고는 제작 단계로 넘기지 않는다.

## 2. 제작 형식 확인

영상 제작 형식은 매우 다양하므로 먼저 비용과 구현 가능성 기준으로 나눈다.

사용자에게 다음 선택지를 제시한다.

- PPT/슬라이드형: 텍스트, 도형, 이미지 중심. Codex/Claude 내부 산출물로 제작 가능.
- 이미지 카드형: 여러 장의 이미지와 자막을 이어 붙이는 형식. 이미지 생성 API가 필요할 수 있음.
- 화면 녹화형: 제품, 웹사이트, 앱 화면을 녹화하고 자막/내레이션을 붙임.
- 실사 촬영형: 사용자가 직접 촬영하고, 제작자는 촬영표와 편집 가이드를 제공.
- AI 영상 생성형: 영상 생성 API가 필요할 수 있고 비용이 발생할 가능성이 높음.
- TTS/더빙형: 음성 API가 필요할 수 있고 사용량 기반 비용이 발생할 수 있음.

## 3. 비용/API 고지

외부 API가 필요한 형식이면 제작 전에 `cost-and-api-disclosure.md` 형식으로 안내한다.

반드시 포함할 것:

- 어떤 서비스가 필요한가
- 왜 필요한가
- 비용이 발생하는 지점
- 무료/로컬 대안이 있는가
- 공식 가격 페이지 또는 대시보드에서 확인해야 할 항목
- 필요한 API 키 이름
- 환경변수 또는 시크릿 저장 위치
- 연결 후 테스트 방법

사용자가 비용/API 연결을 이해하고 승인하기 전에는 외부 API 기반 제작을 진행하지 않는다.

## 4. 스타일 세팅

제작 전에 사용자의 선호 스타일을 확정한다.

질문 예시:

- 화면은 미니멀한가, 역동적인가, 정보 밀도가 높은가?
- 자막은 짧고 강한 문장인가, 설명형 문장인가?
- 색감은 차분한가, 밝고 에너지 있는가?
- 내레이션은 필요한가? 필요하다면 직접 녹음인가, TTS인가?
- 썸네일은 얼굴/제품/큰 문구/비교형 중 어떤 방향인가?

스타일 세팅이 끝나면 `style-preset.md`로 저장한다.

## 5. 제작 초안 생성

확정 원고와 스타일 세팅을 바탕으로 제작 초안을 만든다.

출력:

- 장면 구성표
- 컷별 화면 내용
- B-roll 또는 이미지 프롬프트
- 자막 문구
- 내레이션/TTS 큐
- 편집 리듬
- 썸네일 방향
- 필요한 파일 목록

## 6. 사용자 컨펌

제작 초안을 만든 뒤 사용자에게 선택하게 한다.

- 이대로 제작/편집 단계로 진행
- 특정 장면만 수정
- 전체 스타일 방향 재선택
- 비용/API 방식 변경

컨펌 전에는 최종 제작물로 확정하지 않는다.

## 7. 피드백 처리

피드백이 명확하면 바로 해당 위치에 반영한다.

피드백이 많거나 추상적이면 선택지를 제시해 방향성을 좁힌다.

예시:

- 더 빠르게: 컷 길이를 줄이고 자막을 짧게 만든다.
- 더 고급스럽게: 색 수를 줄이고 여백과 느린 전환을 늘린다.
- 더 친근하게: 설명 자막과 예시 화면을 늘린다.
- 더 판매 중심으로: 문제-해결-CTA 구조를 강화한다.

## 8. 선호 규칙 저장

반복 적용할 만한 피드백은 바로 규칙으로 저장하지 않는다.
반드시 사용자에게 묻는다.

> 앞으로 이 채널의 영상 제작에서는 이 스타일을 기본값으로 적용할까요?

사용자가 동의하면 `production-preferences.md`에 저장한다.

## 9. 채널 매니저에게 전달

사용자가 제작 초안을 컨펌하면 채널 매니저에게 다음을 넘긴다.

- 확정 제작 방향
- 썸네일 방향
- 제목/썸네일 실험 후보
- 설명란에 반영할 핵심 메시지
- Shorts 또는 클립 재활용 포인트
- 제작 중 발견한 주의사항
