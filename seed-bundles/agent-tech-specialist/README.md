# 에이전트 기술 전문가 번들

처음 사용하는 분은 [처음사용하기.md](./처음사용하기.md)부터 읽으세요.

에이전트 기술 전문가는 AI력 사무소의 에이전트 제작 팀이 최신 제작 기술을 놓치지 않도록 기술 동향을 추적·자문하는 AI 직원이다.

에이전트 제작 기술(MCP 생태계, Claude Code/Codex 런타임 기능, 모델 업데이트, 에이전트 설계 패턴, 도구 연결·권한 기술)은 매주 바뀐다. 제작 팀이 각자 업무를 하는 동안 신기술을 아무도 챙기지 않으면, 팀의 규칙과 스킬은 조용히 낡는다. 에이전트 기술 전문가는 쏟아지는 기술 소식 중 팀 업무에 실제로 영향을 주는 것만 걸러 브리핑하고, 제작 과제에 맞는 기술을 자문하며, 낡아진 팀 번들의 규칙을 갱신할 `보수 교육 제안서`를 만든다.

핵심은 '뉴스 요약기'가 아니라 '팀의 참모'다. 기술 소식을 다 전하지 않고, 팀 업무에 유의미한 것만 신호 강도와 성숙도를 붙여 전달한다. 확인하지 못한 정보는 사실처럼 단정하지 않는다.

## 파이프라인 위치

```txt
직무 분석가 → 에이전트 메이커 → 에이전트 훈련소 → 실전 투입
                        ↑
              [에이전트 기술 전문가]
         (팀 전체를 지원하는 기술 참모)
```

특정 단계를 맡지 않고 파이프라인 전체를 지원한다. 훈련소가 **신임 에이전트의 직무 지식**을 적재한다면, 기술 전문가는 **팀 자체의 제작 기술**을 최신으로 유지한다.

## 사용 목적

- 에이전트 제작에 영향을 주는 기술 변화를 놓치지 않는다.
- 소음(팀 업무와 무관한 뉴스)을 걸러내고 유의미한 신호만 브리핑한다.
- 특정 에이전트 제작 과제에 맞는 MCP·도구·설계 패턴을 자문한다.
- 신기술이 팀 번들(분석가/메이커/훈련소)의 규칙을 낡게 만들면 보수 교육 제안서를 만든다.
- 모든 판단에 신호 강도(강/중/약)와 출처를 정직하게 표기한다.

## 3가지 산출물

1. **기술 동향 브리핑**: 팀에 유의미한 기술 변화만 골라 신호 강도·성숙도·팀 적용 가능성과 함께 정리한 브리핑.
2. **기술 자문서**: 특정 제작 과제에 대해 직무의 도구 요구를 도출하고, 후보 기술을 비교해 추천안 2~3개를 제시하는 자문.
3. **보수 교육 제안서**: 신기술로 낡아진 팀 번들의 skills/memory 업데이트 후보를 diff 형태로 제안하는 문서. 자동으로 고치지 않고 항상 제안서로 만든다.

## 포함 파일

- `AGENTS.md`: 폴더 연결 진입점 (런타임이 폴더를 열 때 자동으로 읽는 지시문 로드 순서 안내)
- `worker/agent.md`: Claude/Codex에 넣어 사용하는 직원 지시문
- `worker/agent.json`: 직원 메타데이터
- `worker/skills/tech-radar-rules.md`: 기술 추적·신호 판정 규칙
- `worker/skills/advisory-rules.md`: 제작 과제 기술 자문 규칙
- `worker/skills/refresher-training-rules.md`: 보수 교육 제안 규칙
- `worker/skills/source-credibility-rules.md`: 기술 동향 소스 공신력 판단 규칙
- `memory/`: 운영 원칙, 사용자 선호, 팀 맥락, 결정 이력, 기술 레이더 누적 기록
- `examples/input/request.md`: 예시 요청
- `examples/output/tech-briefing.md`: 예시 기술 동향 브리핑
- `docs/`: 비용, 보안, 한계, 제작 기록

## 사용 방법

1. `worker/agent.md` 내용을 Claude 프로젝트 지침, Codex 지침, 또는 사용하는 에이전트 런타임의 시스템 지침에 넣는다.
2. 팀 맥락(사용 런타임, 만드는 번들 형태, 관심 기술 영역)을 알려준다. `memory/PROJECT.md`에 채워두면 매번 말하지 않아도 된다.
3. 원하는 업무를 요청한다: 동향 브리핑 / 특정 제작 과제 기술 자문 / 팀 번들 보수 교육 점검.
4. 브리핑·자문·제안서를 검토하고, 채택할 항목만 승인한다.
5. 보수 교육 제안서가 승인되면 대상 번들의 담당 직원(또는 사용자)이 실제 수정을 반영한다. 기술 전문가가 직접 다른 번들을 고치지 않는다.
6. 추적할 가치가 있는 기술은 `memory/TECH-RADAR.md`에 기록 후보로 남겨 다음 브리핑에서 이어서 추적한다.

## 예시 입력

```txt
이번 주 에이전트 기술 동향 브리핑해줘. 우리 팀은 Claude Code에서 폴더형 번들을 만들어.
```

## 주의

이 번들은 호스팅 서비스가 아니다. 사용자는 자신의 Claude, Codex, OpenAI API, 또는 기타 모델 환경에서 직접 실행한다. 웹 검색이 불가능한 환경에서는 최신 동향을 단정할 수 없어 "재확인 필요" 표시가 붙는다. 기술 채택 결정, 팀 번들 수정 확정, 비용 발생 도구 도입은 사용자가 최종 책임진다.
