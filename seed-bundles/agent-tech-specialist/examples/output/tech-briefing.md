# 기술 동향 브리핑 (예시)

> 이 문서는 브리핑 **형식을 보여주는 예시**다. 아래 항목은 실제 최신 동향이 아니라 형식 설명용 가상 항목이며, 실제 브리핑은 당일 웹 검색으로 확인한 내용과 실제 출처 링크로 작성된다.

## 1. 요약

이번 주는 팀에 유의미한 변화 2건을 확인했다. 하나는 팀 번들의 도구 연결 규칙에 영향을 줄 수 있는 런타임 권한 설정 변화(신호 강도 강), 하나는 아직 추적만 필요한 MCP 생태계 움직임(신호 강도 중)이다. 즉시 반영이 필요한 것은 없고, 1건은 보수 교육 제안 검토 후보다.

> 검색 상태: 당일 웹 검색으로 확인함. (검색 불가 환경이었다면 여기에 "검색 불가 — 전 항목 재확인 필요"로 표시된다.)

## 2. 유의미한 동향

### (예시 항목 A) 런타임의 도구 권한 설정 방식 변경

- 범주: 도구 연결·권한 기술
- 신호 강도: 강 (공식 릴리스 노트로 확인)
- 성숙도: 안정
- 팀에 왜 중요한가: 팀 번들이 쓰는 권한 설정 파일의 권장 형식이 바뀌면, 에이전트 메이커의 도구 연결 규칙이 낡은 형식을 안내하게 된다.
- 권장 액션: 에이전트 메이커 번들의 도구 연결 규칙과 대조 → 차이가 확인되면 보수 교육 제안서 작성.
- 팀 적용 판정: 즉시 검토
- 출처: (공식 릴리스 노트 링크. 날짜.) [A]

### (예시 항목 B) 특정 직군용 MCP 서버 공개

- 범주: MCP 생태계
- 신호 강도: 중 (개발사 블로그 + 독립 소스 1곳, 공식 문서는 아직 초안)
- 성숙도: 조기 채택
- 팀에 왜 중요한가: 해당 직군 에이전트 제작 시 도구 인터뷰의 후보군이 넓어진다. 지금 만드는 번들에는 해당 직군이 없어 즉시 액션은 없다.
- 권장 액션: 없음 — TECH-RADAR에 등록하고 성숙도 변화를 추적.
- 팀 적용 판정: 추적만
- 출처: (개발사 블로그 링크. 날짜.) [B]

## 3. 계속 추적 중

- (예시) 새 에이전트 간 통신 사양 예고 — 신호 강도 약(단일 커뮤니티 소스). 공식 발표 전까지 판단 보류. [C]

## 4. 버린 소음

- (예시) 모 플랫폼의 마케팅성 발표 — 구체적 사양 변화 없음, 팀 런타임과 무관.

## 5. TECH-RADAR.md 업데이트 후보

- 추가: 예시 항목 B (범주 MCP 생태계 / 신호 중 / 조기 채택 / 추적만)
- 이력 추가: 예시 항목 A — "YYYY-MM-DD: 권장 형식 변경 확인(릴리스 노트)"
