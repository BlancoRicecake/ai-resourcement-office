# Limitations

- 이 번들은 호스팅 서비스가 아니다. 사용자의 모델 환경에서 직접 실행한다.
- 웹 검색이 불가능한 환경에서는 최신 동향을 확인할 수 없어 모든 항목에 "재확인 필요"가 붙는다. 이때 브리핑은 방향 제시 수준에 그친다.
- 브리핑은 실행 시점 기준이다. 에이전트 기술은 빠르게 바뀌므로 시간이 지난 브리핑은 재확인이 필요하다.
- 추천 도구를 직접 설치·실행해 검증하는 것은 기본 범위가 아니다. 자문은 공개 정보 기반 비교이며, 실제 검증은 파일럿 적용에서 이뤄진다.
- 보수 교육 제안서는 제안일 뿐이다. 대상 번들 반영, 버전 올림, 실제 수정은 사용자(또는 담당 직원)가 수행한다.
- 신임 에이전트의 직무 지식 훈련(도메인 지식 적재)은 하지 않는다. 그것은 에이전트 훈련소의 역할이다.
- 판단 품질은 `memory/PROJECT.md`의 팀 맥락 구체성에 좌우된다. 팀 런타임·번들 형태가 비어 있으면 "팀 적용 가능성" 판정이 일반론에 그친다.
- 기술 채택·비용 발생 결정의 최종 책임은 사용자에게 있다.
