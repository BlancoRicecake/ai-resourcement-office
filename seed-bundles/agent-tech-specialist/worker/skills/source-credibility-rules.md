# Source Credibility Rules — 기술 동향 소스 공신력 판단 규칙

기술 전문가는 아무 소식이나 근거로 삼지 않는다. 기술 동향 소스의 공신력을 판단하고, 루머·과장을 걸러낸다.

## 소스 등급

- **A (공식)**: 공식 문서, 릴리스 노트, 체인지로그, 사양 문서, 개발사 공식 발표. 신호 강도 "강"의 근거.
- **B (준공식)**: 개발사 엔지니어링 블로그, 메인테이너의 공개 발언, 공신력 있는 기술 매체의 확인 보도. 신호 강도 "중"의 근거 (복수 소스 일치 시).
- **C (커뮤니티)**: 포럼·소셜 미디어 글, 개인 블로그, 벤치마크 주장, 예고·소문. 신호 강도 "약". 단독으로는 브리핑 항목의 근거가 되지 않는다.

## 필터링 기준

| 기준 | 질문 |
| --- | --- |
| 공신력 | 발표 주체가 그 기술의 개발·유지보수 당사자인가? |
| 최신성 | 발행일이 언제인가? 그 후 뒤집힌 발표는 없는가? |
| 일관성 | 독립된 복수 소스가 같은 내용을 말하는가? |
| 구체성 | 버전·사양·날짜가 명시되어 있는가, 인상 평가뿐인가? |
| 이해관계 | 소스가 그 기술을 팔거나 홍보할 이해관계가 있는가? |

## 적용 규칙

- C급 단독 주장은 "계속 추적 중"에만 두고, 권장 액션·보수 교육 제안의 근거로 쓰지 않는다.
- B급은 서로 독립된 2개 이상이 일치할 때 신호 강도 "중"으로 올린다.
- 성능 수치·채택률 주장은 A급 출처가 없으면 숫자를 옮기지 않고 "주장 있음(미확인)"으로 표기한다.
- 발행일을 확인한다. 에이전트 기술은 몇 달 전 글도 낡았을 수 있다 — 오래된 소스는 당일 검색으로 현재도 유효한지 재확인한다.
- 과장 신호를 의심한다: 구체적 사양 없이 수식어만 큰 발표, 출처를 서로 돌려 인용하는 기사들, 삭제·정정된 원 소스.
- 웹 검색이 불가능한 환경이면 소스 확인 자체가 불가하므로, 모든 항목을 "재확인 필요"로 표기하고 등급을 부여하지 않는다.

## 출처 표기 형식

```txt
- 소스명. 제목/내용 요지. 날짜. URL. [등급 A/B/C]
```

원문 장문 인용은 저장하지 않는다. 위 한 줄 표기와 자기 언어의 요지만 남긴다.
