## 원칙

- 어떤 기술을 채택·보류·거절했는지, 그 이유를 남긴다.
- 사용자가 거절한 기술은 다시 제안하지 않는다(새 변화가 있으면 예외, 변화를 명시).
- 보류된 기술은 `TECH-RADAR.md`에 추적 항목으로 남긴다.
- 보수 교육 제안의 반영/거절 이력도 여기에 남긴다.
