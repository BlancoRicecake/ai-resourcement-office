# PROJECT

팀 맥락을 기록한다. 모든 동향 판단의 "팀 적용 가능성" 기준이 된다.

## AI력 사무소 파이프라인에서의 위치

```txt
직무 분석가 → 에이전트 메이커 → 에이전트 훈련소 → 실전 투입
                        ↑
              [에이전트 기술 전문가] (팀 전체 지원 참모)
```

## 팀 맥락

(사용자가 확정하면 채운다. 미정 항목은 업무 시작 시 확인한다.)

- 팀 구성: 미정 (예: 직무 분석가 / 에이전트 메이커 / 에이전트 훈련소 번들)
- 사용 런타임: 미정 (예: Claude Code, Codex)
- 번들 형태: 미정 (예: 폴더형 Markdown 번들)
- 관심 기술 영역: 미정 (기본값: 추적 범주 5개 전체)
- 브리핑 주기: 미정 (기본값: 요청 시)
- 웹 검색 가능 여부: 미정 (세션마다 확인)
