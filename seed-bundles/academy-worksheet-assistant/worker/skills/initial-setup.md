# 최초 설정

## 시작 조건

`config/profile.json`이 없거나 사용자가 과목·교육과정을 바꾸려는 경우 적용한다.

## 초보자 대화 원칙

- 먼저 `제가 한 번에 하나씩 물어볼게요. 모르면 잘 모르겠다고 답해도 됩니다.`라고 안내한다.
- 개발자 용어, JSON, 경로, 스키마, pending 같은 내부 표현을 사용자에게 먼저 보여주지 않는다.
- 질문은 한 메시지에 하나만 한다. 선택지가 도움이 되면 2~4개의 쉬운 예를 함께 제시한다.
- 사용자가 모르는 항목은 억지로 결정하게 하지 않고 `미정`으로 둔다.
- Python 설치나 명령어는 초기 설정에서 요구하지 않는다. 첫 PDF 작업을 시작할 때 필요 여부를 확인한다.

## 질문 순서

1. 사용할 과목
2. 학교급, 기본 학년, 기준 시험 연도
3. 교육과정 이름을 아는지. 모르면 자료를 `user-materials/`에 넣도록 안내
4. 출판사, 교재명, 단원 배열 자료
5. 입력 형식(PDF만 지원)과 출력 형식(HWP/PDF)
6. 학원 레이아웃·해설 수준·금지 표현

질문이 끝나면 사용자의 답변을 쉬운 표로 요약하고 `이 내용으로 저장할까요?`라고
확인한다. 승인 전에는 파일을 쓰지 않는다. 승인 후 에이전트가 `setup.py`와 같은
스키마로 `config/profile.json`, `memory/USER.md`, `memory/PROJECT.md`를 직접
작성한다. `python setup.py`는 명령어 사용을 원하는 고급 사용자에게만 안내한다.

## 완료 판정

- 필수 항목이 채워지면 `setup_status=complete`다.
- 공식 교육과정과 과목 지식이 사용자 승인되기 전까지 `knowledge_status=pending`이다.
- setup 완료와 knowledge ready를 혼동하지 않는다.
- HWP는 Windows+한컴오피스가 없으면 선택할 수 없다고 안내한다.
