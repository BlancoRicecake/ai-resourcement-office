# Skill: 법률 문서 초안 작성

법률 문서 초안 직원이 서비스 정보를 받아 초안을 만들 때 따르는 절차.

## 절차

1. 서비스명, 설명, 운영 주체, 유료 여부, 개인정보 수집 항목을 확인한다.
   불명확하면 사용자에게 먼저 묻는다 — 특히 유료 여부와 개인정보 수집
   여부는 조항 구성이 달라지므로 추측하지 않는다.
2. 미니 SaaS CLI로 초안을 생성한다:

   ```bash
   python app/app.py --cli <입력.md> -o <문서.md>
   ```

3. 생성된 초안을 읽고 다음을 점검한다:
   - 최상단에 "법률 자문 아님 + 변호사 검토 필요" 고지가 있는가
   - 서비스 실정에 맞지 않는 조항이 섞이지 않았는가
     (무료 서비스에 결제 조항, 미수집 서비스에 수집 조항 등)
   - 개인정보처리방침이 필수 기재사항 구조(수집 항목·목적, 보유 기간,
     제3자 제공, 위탁, 파기, 정보주체 권리, 안전성 확보조치, 보호책임자)를
     갖췄는가
   - [대괄호] 미확정 값이 확정된 것처럼 쓰이지 않았는가
4. "변호사 검토 후 게시해야 한다"는 안내와 함께 초안을 전달한다.

## 주의

- 사용자가 게시 가능 여부를 물으면 항상 전문가 검토를 권한다. 단정하지 않는다.
- 대한민국 법령 기준이다. 해외 사용자 대상 서비스(GDPR, CCPA 등)는 별도
  검토가 필요함을 알린다.
- 법령 개정 가능성을 항상 언급한다.
- API 키가 없으면 mock 모드(템플릿)로 실행된다. 결과 전달 시 모드를 명시한다.
