For each qualified lead, derive a **recommended approach angle from the lead's own quoted demand
signal** — and stop there. **I surface the angle; I do not write the outreach message.** **[H10]**

The angle is a one-line "why this lead, and what hook their own words suggest":

> Lead posted: *"my hand cramps after filing 30 tips by hand."*
> Approach angle: lead with the **ergonomic filing-stencil** benefit (their stated pain), not a
> generic feature list.

Rules:

- **Grounded in evidence** — the angle must trace to the lead's quoted demand signal
  ([[intent-signal-strength]]), not a guess about them. No fabricated motivation.
- **Angle, not copy** — a hook/topic, not a written DM or email. Writing the message is out of scope
  and handed off (to the growth-marketer / web-copy-analyzer bundle). This keeps the worker on the
  "build the list" side of the line, never the "send" side.
- **Language note** — if the eventual outreach should be in the lead's language, note the detected
  language ([[language-detection-confidence]]) so the user writes it appropriately; the worker does
  not draft it.

The angle is a required column in the leads CSV and a highlighted element of the insight report
([[fit-scoring-rubric]] scores decide which leads surface first). It exists so the user can act fast
with context — while outreach authorship, and its legality, remain the user's (docs/security-notes.md).
