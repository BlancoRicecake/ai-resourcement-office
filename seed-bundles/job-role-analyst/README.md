# 직무 분석가 번들

처음 사용하는 분은 [처음사용하기.md](./처음사용하기.md)부터 읽으세요.

직무 분석가는 사용자의 에이전트 아이디어를 실제 사회에 존재하는 직무 구조로 해부해, 에이전트 제작 전 단계의 설계서를 만드는 AI 직원이다.

## 사용 목적

- "이런 일을 하는 에이전트를 만들고 싶다"는 요청을 실제 직무로 해석한다.
- 해당 직무의 목표, 책임, 업무 범위, 산출물, 협업 직무를 정리한다.
- 사용자가 직접 컨펌해야 하는 시점과 추가로 만들 에이전트 후보를 제안한다.
- 바로 에이전트를 제작하지 않고, 먼저 직무 기반 설계도를 완성한다.

## 포함 파일

- `AGENTS.md`: 폴더 연결 진입점 (런타임이 폴더를 열 때 자동으로 읽는 지시문 로드 순서 안내)
- `worker/agent.md`: Claude, Codex 등에 넣어 사용할 수 있는 직원 지시문
- `worker/agent.json`: 직원 메타데이터
- `examples/input/request.md`: 예시 요청
- `examples/output/job-role-blueprint.md`: 예시 산출물
- `docs/`: 비용, 보안, 한계, 제작 기록
- `memory/`: 운영 원칙, 사용자 선호, 프로젝트 맥락, 설계 기준, 설계서 이력을 저장하는 메모리 파일

## 사용 방법

1. `worker/agent.md` 내용을 Claude 프로젝트 지침, Codex 지침, 또는 사용하는 에이전트 런타임의 시스템 지침에 넣는다.
2. 자연어로 만들고 싶은 에이전트를 설명한다.
3. 직무 분석가가 만든 `직무 기반 에이전트 설계서`를 검토한다.
4. 설계서의 확인 질문과 추가 에이전트 후보를 바탕으로 다음 직원 제작 범위를 정한다.

## 예시 입력

```txt
유튜브 작가 에이전트를 만들고 싶어.
```

## 주의

이 번들은 호스팅 서비스가 아니다. 사용자는 자신의 Claude, Codex, OpenAI API, 또는 기타 모델 환경에서 직접 실행한다. 최신 시장 정보나 특정 회사의 실제 직무 체계가 필요한 경우에는 별도 조사가 필요하다.
