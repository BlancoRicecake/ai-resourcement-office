# DESIGNS

직무 분석가가 작성한 직무 기반 에이전트 설계서 이력과 배운 점을 기록하는 파일이다.

## 기록 원칙

- 설계서가 실제 번들 제작(에이전트 메이커)과 훈련(에이전트 훈련소)으로 이어졌는지 추적한다.
- 좋은 번들, 좋은 에이전트로 이어진 설계 패턴과 그렇지 않은 패턴을 구분해 남긴다.
- 사용자 피드백으로 수정된 설계 항목은 다음 설계에 반영할 교훈으로 기록한다.

## 기록 형식

```md
## YYYY-MM-DD 설계서 이름

- 만들려는 에이전트:
- 실제 직무 매핑:
- 핵심 설계 결정:
- 사용자 컨펌 지점 설계:
- 후속 단계 결과 (에이전트 메이커/훈련소):
- 유효했던 설계 패턴:
- 배운 점:
- 다음 설계 반영:
```

## 설계 기록

- 아직 기록된 설계서 없음.
