# Limitations

- **판단 품질은 호스트 모델에 달려 있다.** `diagnose_section`/`rewrite_section`은
  prep 툴이라 LLM을 호출하지 않고 판단 재료(payload)만 반환한다. 실제
  진단·리라이트 문장의 질은 이 payload를 읽는 코딩 에이전트/MCP 클라이언트
  모델의 역량에 좌우된다 — 약한 모델은 휴리스틱 라벨을 붙이지 못하거나
  범용 조언으로 후퇴할 수 있다.
- **SPA·JS 렌더링 의존 페이지는 서버사이드 HTML만으로 부족할 수 있다.**
  `fetch_page`는 스크린샷/브라우저 렌더링에 하드 의존하지 않고 HTML 파싱을
  기본으로 한다 — `parse_quality`가 `sparse`/`empty`로 나오면 브라우저에서
  렌더된 HTML을 직접 복사해 붙여넣어야 한다(로컬 Chromium 렌더 옵션은 감지
  시에만 보조로 제안된다).
- **영어 의존 가독성 지표는 한국어 등 비영어 페이지에 그대로 적용되지
  않는다.** `readability_scorecard`의 가독성 grade_level·we/you 비율·전문용어
  밀도 3종은 영어 기준 지표라, 감지 언어가 영어가 아니면
  `english_dependent_metrics_applicable:false`로 참고용 게이팅된다. 구조
  체크리스트·CTA 인벤토리·제목 길이 같은 언어 무관 지표만 그대로 신뢰할 수
  있다.
- **타깃 페르소나가 반드시 필요하다.** 타깃 페르소나 없이는 진단을 시작하지
  않는다 — 빠른 스코어만 보고 싶어도 페르소나 인터뷰(또는 `memory/PERSONAS.md`에서
  재사용) 단계를 건너뛸 수 없다.
- **실제 사용자 테스트를 대체하지 않는다.** 페르소나 기반 시뮬레이션과
  휴리스틱 귀속은 근거 있는 추정이지, 실측 클릭률·전환율·A/B 테스트 결과가
  아니다. 산출물은 의사결정 보조 자료이며 최종 판단은 사용자의 몫이다.
- **팩트를 창작하지 않지만, 검증도 하지 않는다.** 없는 수치·전환율·법적
  주장은 생성하지 않도록 지시되어 있지만("출처 필요" 플래그만 단다), 페이지에
  이미 있는 과장·허위 카피 자체의 사실 검증은 하지 않는다.
- **경쟁사·복수 페이지 비교, 결과 파일 영속 저장/내보내기, 결과 공유는 범위
  밖이다.** 리포트는 세션 내 표시용이며, 필요하면 사용자가 직접
  복사·저장해야 한다.
