# Cost Guide

## 결정론 툴 — $0

결정론 분석 도구 6종(`fetch_page`/`parse_sections`/`readability_scorecard`/
`diagnose_section`/`rewrite_section`/`compare_report`)은 전부 로컬 Node.js
연산이다. API 키가 없고, 어떤 외부 LLM도 호출하지 않는다. 몇 번을 실행하든
비용은 0원이다.

## 판단은 호스트 모델 토큰 비용 — prep-tool 패턴

`diagnose_section`/`rewrite_section`은 **prep 툴**이다 — LLM을 호출하지 않고
"섹션 텍스트 + 페르소나 + 결정론 신호 + 판단 프레임"을 묶은 payload만
반환한다. 실제 진단·리라이트 판단은 이 payload를 받은 **당신의 코딩
에이전트/MCP 클라이언트(호스트 모델)**가 수행하며, 그 추론에 드는 토큰
비용은 그 모델의 요금제를 따른다 — 이 번들이 별도로 청구하지 않는다.

## 대략적인 가이드

| 작업 | 대략적인 컨텍스트 규모 | 비고 |
| --- | --- | --- |
| 페르소나 인터뷰(1회) | 짧은 대화 몇 턴 | 최초 1회, 이후 재사용 |
| above-the-fold 5초 진단 | 섹션 1~2개 payload | 가장 저렴한 단위 |
| 페이지 전체 섹션별 진단 | 섹션 수만큼 payload(보통 5~15개) | 페이지 길이에 비례 |
| 미스 섹션 리라이트 | 미스로 표시된 섹션만 | 진단 후 서브셋이라 전체 재진단보다 저렴 |
| 재검 루프(before/after 재제출) | 진단+리라이트 반복 | 반복 횟수만큼 누적 |

- 이미 쓰고 있는 코딩 에이전트/구독 요금제 안에서 도는 경우가 많아, 별도
  API 사용량으로 체감되지 않을 수 있다.
- 대형 페이지(섹션 수 많음)나 재검 루프를 여러 번 도는 경우 호스트 모델의
  컨텍스트/토큰 사용이 늘어난다 — `fetch_page`의 `max_bytes` 상한(기본
  5MB)과 above-the-fold 우선순위(§③)가 자연스러운 상한 장치다.

## 권장 사항

- 먼저 `examples/`의 샘플 페르소나·HTML로 전체 흐름을 무료로 확인한다
  (결정론 툴만으로도 `parse_sections`/`readability_scorecard` 결과를 볼 수 있다).
- above-the-fold 진단만 먼저 돌려보고, 필요할 때만 섹션 전체로 확장한다.
- 호스트 모델 제공자의 사용량 대시보드에서 한도를 설정해둔다.
