# COPY-PATTERNS

반응이 좋았던 카피 패턴(헤드라인·CTA·구조), 페르소나별로 잘 통한 표현, 피드백에서 배운 점을 누적하는 직무별 파일이다. 이 파일은 **지식의 인박스**다.

> 여기에 쌓인 항목 중 **일반화 가능한 카피 원칙**이 확인되면 `memory/knowledge/`의 노드로
> **승격**한다(새 `<node-id>.md` + `INDEX.md`에 한 줄 추가). 승격한 원 항목에는
> `→ [[node-id]]` 역참조를 남긴다. 반복 확인된 원칙은 새 노드를 만들지 말고 기존 노드의
> `confidence`를 올린다. 노드 추가·승격도 자동 확정하지 않고 `메모리 업데이트 후보`로 제안한다.
> (사용자·브랜드 종속 선호는 USER.md/DECISIONS.md로, 범용 카피 원칙은 여기 인박스 → 지식 노드로 분리한다.)

## 기록 원칙

- 진단·리라이트를 마칠 때마다 어떤 헤드라인/CTA/구조가 채택·거부됐는지 남긴다.
- 반응 근거(사용자 피드백·채택 여부)가 있으면 함께 기록한다. 근거 없는 패턴은 `검증 필요`로 표시하고 확정 패턴으로 올리지 않는다.
- 후기·증언 원문이나 미공개 매출/전환 실측치는 기록하지 않는다 — 패턴(아이디어)만 재서술한다.
