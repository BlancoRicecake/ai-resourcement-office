# COPY-PATTERNS

반응이 좋았던 카피 패턴(헤드라인·CTA·구조), 페르소나별로 잘 통한 표현, 피드백에서 배운 점을 누적하는 직무별 파일이다.

> 런타임 경로에서는 일반화 가능한 도메인 패턴이 성장 레이어의 지식 그래프
> (`~/.web-copy-analyzer/knowledge/`, `learn_knowledge`)에 누적되고, 반복 절차는 `save_workflow`로 증류된다.
> 순수 폴더 읽기 경로에서는 여기에 `COPY-PATTERNS.md 후보`로 제안해 반영한다.
> (사용자·브랜드 종속 선호는 USER.md/DECISIONS.md로, 범용 카피 원칙은 여기로 분리한다.)

## 기록 원칙

- 진단·리라이트를 마칠 때마다 어떤 헤드라인/CTA/구조가 채택·거부됐는지 남긴다.
- 반응 근거(사용자 피드백·채택 여부)가 있으면 함께 기록한다. 근거 없는 패턴은 `검증 필요`로 표시하고 확정 패턴으로 올리지 않는다.
- 후기·증언 원문이나 미공개 매출/전환 실측치는 기록하지 않는다 — 패턴(아이디어)만 재서술한다.
