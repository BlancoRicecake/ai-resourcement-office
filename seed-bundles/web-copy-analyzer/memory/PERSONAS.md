# PERSONAS

타깃 페르소나를 누적하는 직무별 파일이다. 진단은 페르소나 없이 성립하지 않으므로(H11),
여기서 활성 페르소나를 고르거나 새로 인터뷰해 추가한다.

## 사용 규칙

- 진단을 시작할 때 여기서 대상 페르소나를 **선택**한다. 맞는 페르소나가 없으면 5속성
  (이름·직무·목표·불안(고통)·구매 트리거·어휘)을 인터뷰해 **새 페르소나 추가 후보**로 제안한다
  (자동 확정 금지 — 사용자 승인 후 아래에 기록).
- `diagnose_section` / `rewrite_section` 도구를 호출할 때는 선택한 페르소나를 **명시적 입력**
  (`persona` 인자, 아래 기록을 `{ name, attributes: { role, goals, pains, vocabulary, buying_triggers } }`
  wire 형식으로 변환)으로 전달한다. 별도 페르소나 저장소(store)는 없다.
- 위생: 실제 고객의 PII(이메일·전화·주민번호)나 미공개 매출/전환 실측치는 페르소나에
  적지 않는다 — 페르소나는 세그먼트 모델(직무·동기·어휘)이지 개인 신상이 아니다.

## 페르소나 기록 형식

각 페르소나는 다음 형식으로 한 블록씩 적는다:

```
### <이름>
- 직무(role): …
- 목표(goals): …
- 불안·고통(pains): …
- 구매 트리거(buying_triggers): …
- 어휘(vocabulary): …
```

## 저장된 페르소나

- 아직 없음. (첫 진단 시 인터뷰해 여기에 추가 후보로 제안한다.)

  참고: 샘플 페르소나 입력 형식은 `examples/input/persona.json` (부트스트랩 창업자).

## 업데이트 규칙

새 페르소나·기존 페르소나 수정은 자동 확정하지 않고 `PERSONAS.md 후보`로 제안한 뒤
사용자 승인 후 반영한다.
