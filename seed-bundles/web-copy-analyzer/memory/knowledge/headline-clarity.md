헤드라인에는 단 하나를 묻는다: **"이 문장만 읽어도 무엇을 파는지 아는가."** 슬로건처럼 읽히거나
분위기만 풍기면 명확성 미스다. 영리함보다 명료성이 우선이다. **[H3]**

고칠 때는 구체성을 요구한다 — 수치·명사·결과를 앞세운다. 단, **없는 수치는 절대 창작하지
않는다**(근거 없으면 "출처 필요" 플래그만). **[H6]**

정보어는 왼쪽·앞으로 당겨 [[f-pattern]] 스캔에 태운다. 기능 서술로 끝나면
[[feature-to-benefit]] 번역으로 혜택을 드러낸다.
