# Skill: 타깃 페르소나 인터뷰

`memory/PERSONAS.md`에 맞는 페르소나가 없을 때(최초 실행 포함) 또는 사용자가 새
페르소나를 정의하려 할 때 따르는 절차. `worker/agent.md` ④ "페르소나 없이 진단
시작 금지" 규칙을 실행 가능한 단계로 풀어놓은 것이다.

> 인터뷰로 확정한 페르소나는 `memory/PERSONAS.md`에 `추가 후보`로 제안해 누적한다
> (자동 확정 금지). 별도 저장소·성장 레이어는 없다.

## 절차

1. `memory/PERSONAS.md`를 열어 기존 페르소나가 있는지 먼저 확인한다. 있으면
   목록을 보여주고 재사용할지 새로 정의할지 물어본다.
2. 새로 정의하는 경우, 다음 5속성을 순서대로 인터뷰한다(도구 `persona` 입력의
   `attributes` 필드와 1:1 대응):
   - **role** — 이 페이지의 타깃 구매자는 어떤 직무·상황인가?
   - **goals** — 이 사람이 이루려는 목표는 무엇인가?
   - **pains** — 이 사람의 불안·고통(구매를 망설이게 하는 이유)은 무엇인가?
   - **buying_triggers** — 무엇을 보면 신뢰하고 구매를 결정하는가?
   - **vocabulary** — 이 사람이 실제로 쓰는 단어·표현은 무엇인가(카피
     리라이트 시 그대로 활용).
3. 응답을 요약해 사용자에게 확인받은 뒤 `memory/PERSONAS.md 추가 후보`로
   제안한다. 동명 페르소나가 이미 있으면 덮어쓸지 다른 이름으로 추가할지 확인한다.
4. 승인되면 `PERSONAS.md`에 기록하고, 무엇을 추가했는지 사용자에게 표시한다.
5. 이후 진단·리라이트는 이 페르소나를 `full-page-diagnosis.md` 절차의 1단계에서
   골라 도구의 `persona` 인자(`{ name, attributes: { … } }`)로 명시 전달한다.

## 주의

- 5속성 중 하나라도 빠진 채로 추측해 채우지 않는다 — 불명확하면 다시 묻는다.
- 인터뷰 중 알게 된 정보 중 PII(실명+연락처 등)나 미공개 매출 수치는
  페르소나의 어떤 필드에도 넣지 않는다(④ 위생 규칙과 동일 기준).
