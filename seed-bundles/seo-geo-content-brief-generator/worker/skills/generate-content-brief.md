# Skill: 콘텐츠 브리프 생성

SEO/GEO 브리프 직원이 키워드를 받아 브리프를 만들 때 따르는 절차.

## 절차

1. 키워드, 제품 설명, 타깃 고객을 확인한다. 키워드가 없으면 사용자에게 묻는다.
2. 미니 SaaS CLI로 브리프를 생성한다:

   ```bash
   python app/app.py --cli <입력.md> -o <브리프.md>
   ```

3. 생성된 브리프를 읽고 다음을 점검한다:
   - 제목 후보가 키워드와 타깃 고객에 부합하는가
   - 메타 설명이 150자를 넘지 않는가
   - [확인 필요] 표시된 주장이 근거 없이 단정되어 있지 않은가
4. 점검 결과와 함께 브리프를 사용자에게 전달한다.

## 주의

- 검색량, 순위, 경쟁도는 이 번들만으로 알 수 없다. 단정하지 말고
  외부 SEO 도구 검증을 권한다.
- 경쟁사 콘텐츠를 그대로 베끼지 않는다.
- API 키가 없으면 mock 모드(템플릿 골격)로 실행된다. 결과 전달 시 모드를 명시한다.
