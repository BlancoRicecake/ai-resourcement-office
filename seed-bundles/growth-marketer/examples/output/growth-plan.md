# 성장 실험 설계서

## 1. 요청 해석

사용자는 신규 B2B SaaS의 무료 체험 신청을 늘리고 싶어 한다. 현재 문제는 단순 노출 부족이 아니라, 타깃 고객이 문제를 인식하고 제품 가치를 이해한 뒤 무료 체험까지 이동하는 퍼널이 약한 상태로 해석된다.

## 2. 수익 시간축 판단

- 분류: 중기 검증형에 가까운 단기 수입 혼합형
- 판단 근거: 사용자는 4주 안에 무료 체험 신청 증가를 원한다. 직접 매출보다 리드/체험 신청이 목표이므로 단기 현금흐름보다는 획득 채널 검증에 가깝다.
- 우선할 전략: 문제 해결형 콘텐츠, 링크드인 배포, 랜딩페이지 CTA 개선, 기존 네트워크 이메일 아웃리치
- 피해야 할 전략: 장기 SEO 콘텐츠만 작성하고 전환 CTA를 두지 않는 것, 검증 전 큰 광고비를 쓰는 것
- 권장 실험 기간: 4주

## 3. 성장 목표 정의

- 1차 KPI: 무료 체험 신청 수
- 보조 KPI: 블로그 방문수, 링크드인 게시물 클릭수, 랜딩페이지 전환율, 이메일 회신율
- 기간: 4주
- 예산: 월 100만 원 이하

가정: 현재 정확한 방문수와 전환율이 제공되지 않았으므로, 첫 1주는 기준 지표 수집을 포함한다.

## 4. 현재 퍼널 가정

| 단계 | 현재 상태 가정 | 문제 가능성 |
| --- | --- | --- |
| 인지 | 타깃 고객이 제품을 잘 모름 | 문제 정의 콘텐츠 부족 |
| 유입 | 블로그 유입이 적음 | 검색/배포 채널 약함 |
| 전환 | 무료 체험 신청이 적음 | 랜딩페이지 메시지 부족 |
| 활성화 | 데이터 없음 | 온보딩 확인 필요 |
| 유지 | 데이터 없음 | v1 범위 밖 |

## 5. 핵심 병목 진단

우선 병목은 `유입 부족`과 `전환 메시지 부족`이다. 블로그가 있지만 유입이 적고, 무료 체험 신청도 거의 없으므로 콘텐츠 배포와 랜딩페이지 메시지를 함께 개선해야 한다.

## 6. 우선 실험 가설

초기 스타트업 PM과 창업자가 겪는 `고객 인터뷰 정리 부담`을 직접적으로 건드리는 문제 해결형 콘텐츠를 만들고, 글 끝과 링크드인 배포문에서 무료 체험 CTA를 명확히 제시하면 무료 체험 신청이 증가한다.

## 7. 실행 채널과 메시지

- 블로그: 문제 해결형 콘텐츠 3개 작성
- 링크드인: 각 콘텐츠를 창업자/PM 관점의 짧은 인사이트 글로 재가공
- 이메일: 기존 리드나 지인 네트워크에 보낼 짧은 소개 메일 작성
- 랜딩페이지: 첫 화면 메시지와 CTA를 무료 체험 중심으로 정리

핵심 메시지:

`고객 인터뷰는 많이 하는데, 팀이 다시 읽고 실행으로 옮기지 못한다면 인터뷰 기록을 요약 가능한 지식 자산으로 바꿔야 한다.`

## 8. 실험 프로세스

| 단계 | 수행 행동 | 입력 | 산출물 | 컨펌 |
| --- | --- | --- | --- | --- |
| 1. 수익 시간축 판단 | 4주 목표가 단기 수입인지 중기 검증인지 분류한다 | 목표, 기간, 예산 | 시간축 판단 | 컨펌 필요 |
| 2. 기준 지표 확인 | 현재 방문수와 신청 수를 정리한다 | 최근 4주 지표 | 기준 지표 표 | 컨펌 필요 |
| 3. 타깃 문제 정리 | PM/창업자의 인터뷰 관리 문제를 문장화한다 | 고객 설명, 기존 메시지 | 문제-가치 메시지 | 컨펌 필요 |
| 4. 콘텐츠 주제 선정 | 블로그 주제 3개를 고른다 | 타깃 문제, 제품 가치 | 콘텐츠 리스트 | 컨펌 필요 |
| 5. 랜딩 CTA 수정안 작성 | 무료 체험 신청 중심 문구를 만든다 | 기존 랜딩페이지 | 헤드라인/CTA 초안 | 컨펌 필요 |
| 6. 배포 문안 작성 | 링크드인과 이메일 문안을 만든다 | 콘텐츠 초안 | 배포문/메일 초안 | 검토 필요 |
| 7. 2주 실행 | 승인된 콘텐츠와 문안을 게시한다 | 확정 산출물 | 게시 결과 | 사용자 실행 |
| 8. 성과 회고 | 지표를 비교하고 다음 실험을 정한다 | 방문수, 클릭, 신청 수 | 회고 리포트 | 컨펌 필요 |

## 9. 사용자 컨펌 지점

- 중기 검증형 전략으로 볼지 확인
- 무료 체험 신청을 1차 KPI로 둘지 확인
- 타깃 고객을 PM/창업자/마케터 중 어디에 우선 둘지 확인
- 핵심 메시지가 제품 실제 가치와 맞는지 확인
- 랜딩페이지 문구 변경 승인
- 링크드인과 이메일 배포 문안 승인
- 예산 사용 여부와 광고 집행 여부 승인
- 이번 실험 결과를 메모리에 저장할지 승인

## 10. 필요한 입력값

- 최근 4주 웹사이트 방문수
- 최근 4주 무료 체험 신청 수
- 기존 랜딩페이지 URL 또는 문구
- 실제 제품 기능 설명
- 고객 인터뷰 또는 고객 불만 사례
- 기존 이메일 리스트 보유 여부
- 브랜드 톤앤매너
- 기존 메모리 파일 내용이 있다면 해당 내용

## 11. 최종 산출물

- 4주 성장 실험안
- 블로그 주제 3개
- 랜딩페이지 헤드라인/CTA 초안
- 링크드인 배포문 3개
- 이메일 아웃리치 초안
- 성과 측정표
- 다음 실험 후보
- 메모리 업데이트 후보

## 12. 성과 측정 기준

| 지표 | 측정 방식 | 성공 기준 |
| --- | --- | --- |
| 무료 체험 신청 수 | 신청 폼 제출 수 | 기준 기간 대비 증가 |
| 랜딩페이지 전환율 | 신청 수 / 방문수 | 기준 전환율 대비 증가 |
| 콘텐츠 클릭수 | 블로그/링크드인 클릭 | 주제별 비교 가능 수준 확보 |
| 이메일 회신율 | 회신 수 / 발송 수 | 관심 고객 문장 수집 |

정확한 성공 기준 수치는 현재 기준 지표가 필요하다.

## 13. 협업 직무 지도

| 구분 | 협업 직무 | 필요한 이유 | 주고받는 산출물 |
| --- | --- | --- | --- |
| 선행 직무 | 고객 리서처 | 고객 문제가 명확해야 메시지가 선명해진다 | 고객 불만, 인터뷰 요약 |
| 병렬 직무 | 콘텐츠 마케터 | 블로그와 링크드인 콘텐츠를 작성한다 | 콘텐츠 브리프, 원고 |
| 병렬 직무 | 랜딩페이지 기획자 | 유입 후 전환 구조를 개선한다 | 헤드라인, CTA, 섹션 구조 |
| 병렬 직무 | 디자이너 | 랜딩과 콘텐츠 소재를 시각화한다 | 디자인 브리프 |
| 후속 직무 | 세일즈 담당자 | 무료 체험 신청자를 상담과 계약으로 연결한다 | 리드 정보, 후속 메시지 |

## 15. 리스크와 검토 필요 항목

- 고객 인터뷰 데이터에 개인정보가 있으면 제거해야 한다.
- 무료 체험 혜택이나 기능 범위를 과장하면 안 된다.
- 광고를 집행할 경우 플랫폼 정책과 예산 승인이 필요하다.
- 실제 전환 판단에는 기준 지표와 추적 설정이 필요하다.
