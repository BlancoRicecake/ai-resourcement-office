# 그로스 마케터 번들

처음 사용하는 분은 [처음사용하기.md](./처음사용하기.md)부터 읽으세요.

그로스 마케터는 제품이나 서비스의 유입, 전환, 재방문, 매출 성장을 개선하기 위해 실험을 설계하고 실행안을 만드는 AI 직원이다.

v1.2부터는 중요한 판단 전에 리서치를 먼저 수행하고, 사업이 단기 수입형인지, 중기 검증형인지, 장기 자산 축적형인지 판단한 뒤, 작업 결과와 리서치 근거를 `memory/` 폴더에 저장할 후보로 정리한다.

## 사용 목적

- 마케팅 목표를 매출과 연결되는 성장 지표로 바꾼다.
- 단기 현금흐름, 중기 채널 검증, 장기 자산 축적 중 현재 우선순위를 판단한다.
- 고객 여정에서 병목이 생기는 지점을 찾는다.
- 광고, 콘텐츠, 랜딩페이지, CRM, 세일즈 핸드오프를 하나의 성장 실험으로 묶는다.
- 실행 전 사용자가 승인해야 할 목표, 예산, 메시지, 채널, 실험 기준을 분리한다.
- 결과 데이터를 바탕으로 다음 실험안과 메모리 업데이트 후보를 제안한다.

## 포함 파일

- `AGENTS.md`: 폴더 연결 진입점 (런타임이 폴더를 열 때 자동으로 읽는 지시문 로드 순서 안내)
- `worker/agent.md`: Claude, Codex 등에 넣어 사용할 수 있는 직원 지시문
- `worker/agent.json`: 직원 메타데이터
- `memory/`: 사용자 선호, 프로젝트 기준, 실험 이력, 결정 사항, 리서치 근거를 저장하는 메모리 파일
- `worker/skills/revenue-horizon-rules.md`: 수익 시간축 판단 규칙`r`n- `worker/skills/research-first-rules.md`: 판단 전 리서치 규칙`r`n- `worker/skills/academic-foundations.md`: 학술/실무 기반 판단 메모
- `examples/input/request.md`: 예시 요청
- `examples/output/growth-plan.md`: 예시 산출물
- `docs/`: 비용, 보안, 한계, 제작 기록

## 사용 방법

1. `worker/agent.md` 내용을 Claude 프로젝트 지침, Codex 지침, 또는 사용하는 에이전트 런타임의 시스템 지침에 넣는다.
2. `memory/` 파일에 기존 프로젝트 정보가 있으면 함께 제공한다.
3. 제품, 타깃 고객, 현재 목표, 사용 가능한 채널, 예산 범위, 기존 성과 데이터를 제공한다.
4. 그로스 마케터가 만든 성장 실험 설계서와 메모리 업데이트 후보를 검토한다.
5. 사용자가 승인한 실험만 실제 집행하고, 승인한 메모리만 파일에 반영한다.

## 예시 입력

```txt
신규 B2B SaaS의 무료 체험 신청을 늘리고 싶어. 예산은 월 100만 원이고, 블로그와 링크드인, 이메일을 쓸 수 있어.
```

## 주의

이 번들은 호스팅 서비스가 아니다. 사용자는 자신의 Claude, Codex, OpenAI API, 또는 기타 모델 환경에서 직접 실행한다. 광고 집행, 예산 사용, 법적 표현 검토, 개인정보 처리, 실제 매출 판단, 메모리 확정은 사용자가 최종 책임진다.
