# Build Log

## Source Problem

에이전트 메이커로 만든 새 에이전트는 `memory/`가 비어 있어 첫 업무 품질이 낮은 콜드 스타트 문제가 있다. 사람 신입의 수습기간처럼, 직무 배경지식을 미리 채워줄 단계가 필요하다.

## MVP Scope

신임 에이전트의 직무에 맞는 공신력 있는 오픈 액세스 지식을 합법적으로 수집하고, 학술 개념을 대상 산업의 실무 지침으로 번역해 신임 에이전트의 memory 파일에 적재하는 훈련 계획서를 만든다.

## Design Inputs From This Session

- 훈련소는 크롤러가 아니라 '사내 지식 번역가'다 — 원문 저장 대신 아이디어만 추출해 재서술(변형적 이용).
- 저작권의 아이디어/표현 이분법을 안전장치의 근거로 삼는다.
- 3단계 파이프라인: 흡수(Reader) → 실무적 합성(Synthesizer) → 적재(Writer).
- 학자가 아니라 실전 투입 가능한 직원을 만든다 — 학술 개념을 반드시 시장·비즈니스 액션으로 번역한다.
- 오픈 액세스·TDM 허용 소스만 사용 (arXiv, Semantic Scholar, OpenAlex, PubMed, DOAJ 등).
- 메모리는 컨펌 기반으로 적재하고 기존 메모리는 덮어쓰지 않는다.

## Pipeline Position

직무 분석가 → 에이전트 메이커 → **에이전트 훈련소** → 실전 투입

## Next Improvements

- 오픈 액세스 API 호출 하네스 추가
- 메모리 적재 자동화 스크립트
- 직무별 표준 커리큘럼 프리셋
- 훈련 완료 후 신임 에이전트 리허설(모의 업무) 단계 연결
