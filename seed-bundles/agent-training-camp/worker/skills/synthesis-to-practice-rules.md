# Synthesis-to-Practice Rules — 학술→실무 번역 규칙

이 단계가 훈련소의 핵심이다. 여기서 '학자'가 아니라 '실무자'가 만들어진다.

## 시니어 디렉터 프롬프트

각 학술 개념을 번역할 때 다음 역할로 사고한다.

> "너는 실무 투입을 앞둔 신입 에이전트를 교육하는 시니어 디렉터다. 방금 읽은 학술 개념을 대상 [산업/비즈니스 모델]에 어떻게 바로 적용할지, 구체적 액션 3가지와 예상 기대 효과로 번역해라. 학술 용어는 실무 비즈니스 용어로 대체해라."

## 지식 항목 표준 형식

각 지식은 다음 5요소로 합성한다.

```txt
### [개념 이름]
- 개념: (한 줄, 자기 언어로 재서술한 핵심)
- 시장 연결: (이 개념이 실제 시장·비즈니스에서 어떻게 작동하는가)
- 실무 액션: (신임 에이전트가 바로 실행할 행동 2~3개)
- 기대 효과: (이 액션이 만드는 결과)
- 주의점 / 출처: (한계·리스크 + 재확인용 짧은 출처)
```

## 번역 원칙

- 학술 용어 → 실무 용어로 치환한다. (예: "흡수 역량(absorptive capacity)" → "외부 정보를 우리 제품 맥락에 맞춰 적용하는 능력")
- 이론을 그대로 옮기지 말고, "그래서 무엇을 하라는 것인가"로 끝맺는다.
- 개념 하나에 실무 액션이 하나도 안 나오면, 그 지식은 아직 배경지식일 뿐 실무 지침이 아니다 → `RESEARCH.md` 후보로만 둔다.
- 대상 산업·비즈니스 모델을 항상 고정한다. 같은 개념도 B2B SaaS와 D2C 커머스에서 액션이 다르다.

## 시간축 구분

- 즉시 적용 가능한 실무 액션과, 장기적으로 쌓아야 하는 자산성 지식을 구분해 표기한다.
- 단기 수익형 / 중기 검증형 / 장기 자산형 중 어디에 해당하는지 태그를 붙이면 신임 에이전트가 우선순위를 판단하기 쉽다.

## 품질 자가 점검

- 학술 미사여구가 남아 있지 않은가?
- 신임 에이전트가 읽고 바로 실행할 수 있는가?
- 대상 산업에 구체적으로 맞춰졌는가?
- 원문 문장을 그대로 옮기지 않고 재서술했는가?
