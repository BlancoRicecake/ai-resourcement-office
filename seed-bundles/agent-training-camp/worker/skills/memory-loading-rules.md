# Memory Loading Rules — 메모리 적재 규칙

메모리는 모델 학습이 아니라, 다음 세션에서 다시 읽어 적용하는 운영 기록이다. 훈련소는 신임 에이전트의 메모리에 배경지식을 채우되, 확정은 사용자 승인 후에만 한다.

## 적재 대상 파일 (신임 에이전트 쪽)

| 파일 | 들어가는 내용 |
| --- | --- |
| `memory/CURRICULUM.md` | Phase 2에서 번역한 실무 지침(직무 배경지식). 훈련소의 최종 산출물. |
| `memory/RESEARCH.md` | 출처·근거(참고문헌만, 원문 없음), 검증 후보 지식 |
| `memory/DECISIONS.md` | 훈련 라운드에서 채택/보류/제외한 지식과 이유 |

신임 에이전트에 해당 파일이 없으면 생성 후보로 제안한다. 도메인 지식 파일 이름이 이미 있으면 그 이름을 따른다.

## 적재 원칙

- 원문 텍스트는 버린다. Phase 2에서 새로 쓴 실무 지침만 적재한다.
- 메모리는 자동 확정하지 않는다. 훈련소는 후보를 제안하고, 사용자가 승인한 것만 파일에 반영한다.
- 기존 메모리를 덮어쓰지 않는다. 충돌 시 병합/교체 후보를 나란히 제시한다.
- 오래되거나 틀린 지식은 교체·삭제 후보로 표시한다.
- 메모리 파일에 API 키, 비밀번호, 개인정보, 원문 발췌를 넣지 않는다.

## 적재 후보 표기 형식

```txt
[적재 후보] 대상 파일: memory/CURRICULUM.md
- 추가할 내용: (요약)
- 기존 내용과 충돌: 없음 / 있음(내용)
- 처리 제안: 추가 / 병합 / 교체
- 근거 출처: (RESEARCH.md에 함께 적재)
```

## 훈련 완료 기준

- 커리큘럼의 각 주제에 실무 지침이 1개 이상 적재됐는가?
- 각 실무 지침에 재확인용 출처가 연결됐는가?
- 원문·긴 인용문이 메모리에 남지 않았는가?
- 고위험 직무면 전문가 검토·규제 확인 문구가 함께 적재됐는가?
- 사용자가 적재를 승인했는가?
