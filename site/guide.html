HIRING GUIDE

AI 직원을 내 환경에 채용하는 법

AI력 사무소의 직원은 완성된 호스팅 서비스가 아니라, 사용자의 도구와 계정에 연결해 바로 일할 수 있도록 준비된 작업 패키지입니다. 먼저 질문에 답하고, 필요한 API만 연결한 뒤, 원하는 AI 실행 환경에서 업무를 맡깁니다.

팀 다운로드 질문 보기

WHY IT TAKES TIME · 채용에 앞서

AI 에이전트는 직원입니다.

1~2주는 시간을 들여 가르쳐야 합니다. 하지만 그 시간이 지나면 당신과 당신의 사업을 아주 잘 아는, 실수를 덜 하며, 연봉협상이 필요 없지만 계속 성장하는 직원이 될 겁니다.

세상은 돈을 넣으면 나오는 자판기가 아닙니다.
씨를 심고 물을 주며 싹이 자라는 걸 지켜보세요.

01

팀을 고른다

필요한 업무에 맞는 AI 직원 팀을 내려받습니다. 한 팀 안에는 여러 직원과 실행 예시가 함께 들어 있습니다.

02

질문에 답한다

처음 실행할 때 직원이 사용할 사이트, 계정, 데이터 위치, 결과 형식을 정합니다.

03

API를 연결한다

답변에 따라 필요한 서비스의 API 키 발급 위치와 권한 범위를 확인합니다.

04

작업을 맡긴다

키와 권한이 준비되면 사용자의 환경에서 바로 업무를 실행합니다.

01 / INTAKE

처음에는 질문부터 받습니다

같은 직원이라도 사용자가 쓰는 사이트와 업무 방식은 다릅니다. 그래서 각 직원은 실행 전에 짧은 질문을 던지고, 답변을 기준으로 필요한 연결만 안내합니다.

사용 도구

어떤 사이트, 툴, 데이터베이스, 파일에서 정보를 가져와야 하나요?

결과 위치

결과물을 Notion, Slack, Google Sheets, 문서, CSV 중 어디에 남길까요?

권한 상태

이미 가지고 있는 API 키, 관리자 권한, 팀 계정이 있나요?

검수 방식

AI가 바로 실행해도 되는 작업과 사람이 확인해야 하는 작업을 나눕니다.

02 / API SETUP

API 키는 사용자의 환경에만 둡니다

AI력 사무소가 사용자의 API 키를 받는 구조가 아닙니다. 각 팀 패키지는 필요한 키 이름과 발급 위치를 알려주고, 사용자는 자신의 로컬 환경이나 배포 환경에 직접 저장합니다.

연결 가이드는 이렇게 구성됩니다

무엇을 연결하는지 먼저 설명합니다.

어디에서 키를 발급하는지 단계별로 적습니다.

어떤 권한만 필요한지 최소 범위로 안내합니다.

.env.example에는 실제 키가 아니라 변수 이름만 둡니다.

03 / FOLDER STANDARD

AI가 읽기 쉬운 폴더로 제공합니다

Claude, Codex, n8n, Hermes 같은 도구가 잘 사용하려면 폴더 안의 역할이 명확해야 합니다. 그래서 모든 직원은 같은 이름의 핵심 파일을 가집니다.

worker-package/
worker-package/ README.md bundle.json .env.example worker/agent.md worker/intake.md worker/connectors/ worker/skills/ worker/harness/ examples/

04 / RUN IT WITH AI

실행 환경별 사용 방식

폴더를 그대로 읽히되, 도구마다 필요한 실행 설명을 따로 제공합니다.

Claude

프로젝트 폴더를 올리고 README, agent.md, intake.md를 기준으로 역할과 질문을 읽게 합니다.

Codex

로컬 폴더를 열어 코드, 스킬, 예시 파일을 함께 읽고 필요한 수정과 실행을 맡깁니다.

n8n

API 연결과 자동화 흐름은 별도 워크플로 설명 또는 JSON 템플릿으로 옮겨 실행합니다.

Hermes

agent.md와 harness 문서를 기준으로 직원 역할, 입력, 출력 형식을 맞춰 사용합니다.

05 / RULE

채용 원칙

좋은 AI 직원은 처음부터 모든 계정을 요구하지 않습니다. 먼저 사용자의 업무를 묻고, 필요한 연결만 좁힌 뒤, API 키는 사용자의 환경에 보관하게 해야 합니다.
