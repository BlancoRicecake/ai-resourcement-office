## 사무소 포지셔닝

AI력 사무소는 다운로드 가능한 AI 워커 패키지를 위한 무료 디렉토리다.
호스팅형 에이전트 마켓플레이스와 다르다 — 에이전트를 대신 실행하지 않고,
사용자 데이터를 저장하지 않고, 모델/API 비용을 대신 내지 않는다. 각 패키지는
사용자가 자기 머신이나 클라우드 계정에 AI 워커를 설치하도록 돕는다.

## 타깃 사용자

솔로 빌더, 소상공인, 마케터, 컨설턴트, AI 에이전트 워크플로를 배우는
개발자, 벤더 종속 없이 재사용 가능한 로컬 AI 워크플로를 원하는 팀.

## 비협상 규칙 (Non-Negotiable Rule)

**AI력 사무소는 워커를 대신 실행하지 않는다.** 사용자가 패키지를 내려받아
본인 환경과 예산으로 실행한다. 모든 API·모델·서버·서드파티 비용은 사용자가
부담한다. 이 문장은 모든 공개 페이지와 README에 명시돼야 한다.

## 공장이 모든 번들에 강제하는 원칙

**1. 운영 유지비는 0원을 지향한다.** 서버가 상시로 무거운 연산을 대신 해주는
구조는 금지한다. 결정론 연산은 사용자의 로컬 머신(CLI/MCP)에서 돈다. 유료로
불가피한 외부 API는 운영자 키가 아니라 사용자 본인의 키(BYOK)를 쓰고, 정말
조정 레이어가 필요하면 아이들 비용 0인 서버리스 무료 티어만 쓴다.

**2. 두 표면(CLI·MCP)은 반드시 하나의 core를 공유한다.** 실제 로직은
`app/core`의 순수 함수/모듈로 떼어두고, CLI·MCP 어댑터는 그 core를 감싸는
얇은 레이어로만 존재한다. core에 없는 로직이 어댑터에 새로 생기면 설계
위반이다. 자연어 에이전트로 쓰는 것은 이 도구 표면 위에 `agent.md` 하나를
얹는 것으로 해결하며, 별도의 세 번째 표면(예: 상시 웹 서비스)을 만들지
않는다.

**3. 폴더-번들이 정본이며 모델·언어에 종속되지 않는다.** "어디서나 붙여넣기"
가능해야 한다 — Claude/Codex/Manus 등 어떤 코딩 에이전트가 폴더를 읽어도
동일하게 동작한다. 언어는 자유이나 TS를 권장한다. Claude Code 플러그인은
`agent.md`로부터 생성되는 여러 어댑터 중 하나일 뿐, 유일한 배포 형태가
아니다.

**4. 웹 쇼룸은 영구/1차 산출물이 아니다.** 예전 하네스 설계(`service-harness-design.md`)는
쇼케이스 웹사이트를 하네스의 상시 산출물로 다뤘지만, 이 결정은 폐기한다.
AI력 사무소 웹사이트는 **카탈로그**로만 남고, 공장은 사이트에 의존하지
않는다. 번들은 웹 UI 없이도 CLI/MCP/agent.md만으로 완결돼야 한다.

**5. 새로 만들기 전에 항상 있는 것부터 확인한다.** 이는 우리 내부 코드·문서에만
적용된다 — `template/`을 복제하고, `memory/`+`AGENTS.md` 학습 루프 패턴은
`seed-bundles/youtube-content-writer/`에서, (필요할 때만) 결정론 코어·성장
레이어·지식 그래프 패턴은 `seed-bundles/web-copy-analyzer/`에서 가져다
쓴다(§`BUILD.md` 참고). 어떤 서비스 아이디어를 고를지는 이 원칙이 아니라
리서치 단계의 기준을 따른다.

**6. 보안은 기본값이다.** 로컬 자격증명은 평문 파일이 아니라 OS 키체인이나
동등 수준으로 다루고(불가능하면 명시), 외부 서비스 권한은 최소 스코프만
요청한다. LLM이 대신 보내는 입력(MCP 툴 호출)도 사람 입력과 동일한 수준으로
검증한다.

**7. 학습 루프 메모리(`memory/`)는 필수, 두-뿌리 지식 그래프는 코드형 번들의
선택적 강화다.** 모든 번들은 `bundle-standard.md` §2가 정의하는 평문
`memory/{MEMORY,USER,PROJECT,DECISIONS,<JOB>}.md` 학습 루프를 갖춰야
한다 — 커스터마이징에서 한 발 더 나가 "쓸수록 자라는 구조" 자체가 산출물의
일부다. 업무를 마치면 배운 점·사용자 피드백·유효했던 패턴을 `메모리 업데이트
후보`로 제안하고, **자동 확정 금지**(사용자 승인만 실제 파일에 반영,
write_approval) 원칙을 지킨다. 위생 필터(API 키·PII·미공개 매출/전환·원문
발췌 금지)는 항상 지킨다. 런타임(`app/`)을 동봉하는 코드형 번들은 그 위에
`knowledge-arch.md`가 정의하는 seed(`worker/knowledge/`, 검증됨·읽기전용) +
learned(`~/.<slug>/knowledge/`, 런타임 축적·업데이트 불변) 2-뿌리 지식
그래프를 **선택적으로** 얹어 연상 조회·검증 게이트·자율 스킬 생성(`save_workflow`)·
정리(consolidation)까지 갖출 수 있다 — 순수-마크다운 번들에는 요구되지
않는다. 그래프를 채택한 번들은 반복 확인이 새 노드가 아니라 `confidence`
상향·중복 병합·모순 플래그로 정리되고, 민감/광범위한 학습 쓰기는 선택적으로
`confidence: pending`으로 대기할 수 있다(자세히는 `knowledge-arch.md` §3,
§3.5). 세션 원시 이력 전체를 뒤지는 에피소드 아카이브는 이 표준의 범위
밖이다 — `decisions.log`/`memory/DECISIONS.md` + 조회로 대체한다.

**7.5. 순수-마크다운 우선.** 기본 번들은 `memory/` + `AGENTS.md` 평문만으로
구성해 런타임 0으로 Claude Code, Codex 등 어떤 코딩 에이전트에서도 폴더를
열자마자 읽힌다. `app/`(결정론 도구)이나 지식 그래프는 결정론 계산·검증
게이트가 실제로 필요할 때만 얹는 선택 강화이며, "번들은 다 코드를 갖춰야
한다"는 가정으로 설계를 시작하지 않는다 — `PRINCIPLES.md` 3번의 "모델·언어에
종속되지 않는다"를 가장 낮은 비용으로 만족시키는 경로가 순수-마크다운이다.

**8. 결정론 도구는 CLI(범용) + MCP(강화)로, Claude 플러그인은 그 위의 얇은
어댑터일 뿐이다.** CLI는 어떤 하네스에서도(스케줄러, 다른 코딩 에이전트)
호출 가능해야 하고, MCP는 자연어 클라이언트에 강화된 경험을 준다. 플러그인은
`agent.md`로부터 생성되는 named agent + MCP 번들 등록의 편의 계층일 뿐, 이
셋 중 아무 것도 core 로직을 새로 갖지 않는다.

**9. 완성된 에이전트는 다른 사람이 설치해 쓸 수 있는 것까지가 산출물이다.**
구현 산출물에는 코드뿐 아니라 README, LICENSE, `.env.example`, 실제 실행으로
얻은 예시(input/output)가 포함된다.

**10. 피드백은 사용자 확인을 받아야만 영구 규칙이 된다.** 세션 중 관찰한
선호나 교정은 곧바로 `agent.md`의 기본 규칙으로 승격하지 않는다. 사용자가
"항상 이렇게 해줘"처럼 확인한 것만 `remember`를 통해 성장 레이어에 영구
기록되고, 그렇지 않은 관찰은 세션 한정으로 취급한다. 팩토리로의 승격(seed로
편입)은 `knowledge-arch.md` §4처럼 반복 확인 후에만, 그리고 팩토리에서
검증 게이트를 통과해야 이루어진다 — 런타임에서 자동 승격하지 않는다.

**11. 게이트는 자기승인하지 않는다.** design 게이트든 verify 게이트든, 만든
사람/세션이 스스로 "통과"를 선언하지 않는다. 작성과 리뷰는 항상 분리된
컨텍스트에서 이뤄진다(`PIPELINE.md`, `verification.md` 참고).
