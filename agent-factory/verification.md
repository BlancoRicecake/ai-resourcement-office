# 검증 (Verification)

`docs/safety.md`(사용자 책임·패키지 규칙·경고 문구)를 흡수하고, 여기에
작성/리뷰 분리 원칙과 학습 루프 메모리 검증 게이트를 더한 문서다.
`PIPELINE.md`의 3단계(verify)가 이 문서의 체크리스트를 통과 기준으로 삼는다.
지식 그래프 게이트(§5)는 **코드형(그래프 강화) 번들에만** 적용된다 —
`bundle-standard.md` v0.3, 부록 A 참고.

## 1. 사용자 책임 (안전 고지 — 모든 번들 README/문서에 반영)

사용자는 스스로 다음을 검토·관리해야 한다:

- API 키
- 모델 사용 비용
- 클라우드/서버 비용
- 서드파티 API 약관
- MCP 권한
- 로컬 파일 접근
- 개인/회사 데이터
- 생성 결과물의 정확성

### 권장 경고 문구

> 이 패키지는 호스팅 서비스가 아닙니다. 코드, 권한, API 비용, 데이터 처리
> 방식을 검토한 뒤에만 실행하세요. 모든 API·모델·서버·서드파티 비용은
> 사용자가 부담합니다.

## 2. 패키지 규칙 (모든 번들이 지켜야 함)

- `.env.example`만 커밋한다. 실제 `.env`는 절대 커밋하지 않는다.
- 필요한 API 키를 설명한다(없으면 "키 불필요"를 명시).
- 예상 비용 요인을 설명한다.
- 데이터가 로컬 밖으로 나가는지 여부를 설명한다.
- MCP를 포함하면 필요한 권한을 설명한다.
- 샘플 input과 output을 제공한다.
- 알려진 한계를 명시한다.

## 3. 작성/리뷰 분리 (자기승인 금지)

- 구현한 사람(세션)이 같은 컨텍스트에서 "검증 통과"를 선언하지 않는다.
- 리뷰는 독립된 컨텍스트/에이전트가 수행한다 — `security-reviewer`(보안),
  `qa-tester`(실행 시나리오), `code-reviewer`(코드 품질), `verifier`(완료
  근거). 최소 1개 이상, 안전·보안이 걸린 변경은 반드시 `security-reviewer`를
  포함한다.
- 리뷰어는 "구현자가 완료라고 주장했다"를 증거로 받아들이지 않는다 — 실제로
  실행하고 관찰한 결과만 증거다.

## 4. 완료 위장 금지 (No-Fake-Completion)

다음은 **블로커**이지 완료 증거가 아니다. 발견 즉시 구현으로 되돌린다:

- `TODO`, `HACK`, `FIXME` 같은 플레이스홀더 주석이 남은 채로 "완료"라고
  보고하는 것
- `test.skip`, `.only`로 비활성화된 테스트
- 스텁(빈 본문) 테스트나 항상 통과하는 가짜 assertion
- 미구현 분기(예: `else { throw new Error("not implemented") }`가 정상
  경로에 남아있음)
- **날조된 예시** — `examples/output/`은 반드시 실제 CLI 실행으로 얻은
  결과여야 한다. 손으로 지어낸 출력은 통과시키지 않는다.

## 5. 지식 그래프 검증 게이트 — 코드형(그래프 강화) 번들에만 적용 (`knowledge-arch.md` §2 반영)

이 게이트는 런타임(`app/`)을 동봉하고 `worker/knowledge/` 그래프 강화를
채택한 번들에만 적용된다. 순수-마크다운 번들(`memory/`만 쓰는 번들)은
이 절을 건너뛴다 — `bundle-standard.md` v0.3 §1, 부록 A 참고.

빌드/테스트로 강제해야 하는 항목:

- [ ] seed의 모든 `[[wikilink]]`가 존재하는 seed 노드 id로 해소된다
      (dangling 링크 0)
- [ ] self-loop가 없다
- [ ] 중복 id가 없다
- [ ] 노드 frontmatter의 `id`가 파일명과 일치한다
- [ ] `graph.json`을 seed에서 재빌드한 결과가 커밋된 `graph.json`과
      동일하다(동기화)

이 게이트를 통과하지 못하면 빌드/테스트가 실패해야 한다 — 사람이 눈으로
확인하고 넘어가는 방식으로 대체하지 않는다.

## 5.5 학습 루프 닫힌 루프 검증 (B/C — memory 학습 루프, `knowledge-arch.md` §3, §3.5)

`memory/` 학습 루프를 쓰는 순수-마크다운 번들에서, 이 절의 "learned 노드 /
`save_workflow` / `confidence: pending`"은 각각 **메모리 업데이트 후보 /
반복 절차의 memory 파일 정리 / 사용자 승인 전 확정 금지(write_approval)**로
치환해 확인한다. 즉 아래 B/C 게이트는 그래프가 없어도 memory 학습 루프
자체의 닫힌 루프 검증으로 그대로 적용된다.

- [ ] **자율 스킬 생성·개선(B)** — `agent.md` §5에 "복잡한 작업을 성공적으로
      마치면 사용자 요청 없이 `save_workflow`로 절차를 증류하고, 기존 스킬과
      겹치면 새로 만들지 않고 개선·갱신한다"는 강령이 실제로 있는지 확인한다.
      말로만 있고 실행에서 매번 새 스킬을 중복 생성하면 미달로 처리한다.
- [ ] **정리(consolidation, C)** — 같은 사실/절차가 반복 확인될 때 새 노드를
      추가하는 대신 기존 노드의 `confidence`가 올라가는지, 근접 중복이
      병합되는지, 모순되는 learned 노드에 `conflict` 플래그가 붙는지
      시나리오로 확인한다. 세션 경계의 가벼운 정리 nudge가 있는지도 본다.
- [ ] **승인 스테이징(선택, C)** — 민감/광범위한 learned 쓰기를 시도했을 때
      즉시 반영되지 않고 `confidence: pending`으로 대기하다가, 사용자 확인
      후에만 활성화되는지 확인한다(이 기능을 채택한 번들에 한함 — 미채택이면
      `docs/limitations.md`에 명시).
- [ ] **에피소드 아카이브(A)는 범위 밖** — 세션 원시 이력 전체를 검색하는
      기능(SQLite FTS5류)의 부재를 결함으로 잡지 않는다. `decisions.log` +
      `search_knowledge`로 대체됐는지만 확인한다.

## 6. 공개 체크리스트 (배포 게이트) — `bundle-standard.md` §6 실행판

배포 전 아래 항목을 전부 체크한다. 하나라도 미달이면 publish 단계로 넘어가지
않는다.

- [ ] `AGENTS.md` 진입점이 존재하고, 로드 순서(`worker/agent.md` → `memory/`
      → `worker/skills/`) · 도구 연결 스코프 · 범위 밖 요청 처리가 모두
      명시돼 있다
- [ ] `memory/` 표준 파일(`MEMORY.md`/`USER.md`/`PROJECT.md`/`DECISIONS.md`
      + 직무별 `<JOB>.md`)이 존재하고, `agent.md`에 학습 루프 규칙(어떤
      memory 파일이 무엇을 담는지 + "완료 후 메모리 업데이트 후보 제안,
      자동확정 금지")이 기술돼 있다
- [ ] `bundle.json`에 `version`(semver, `slug` 바로 다음)이 존재한다
- [ ] README·`.env.example`·LICENSE가 존재하고, 실제 키/시크릿이 커밋되지
      않았다
- [ ] 샘플 input **및 실제 실행으로 얻은** output이 존재한다(§4 참고, 날조
      금지)
- [ ] `docs/cost-guide.md`, `docs/security-notes.md`, `docs/limitations.md`가
      존재하고 내용이 채워져 있다(빈 스켈레톤 상태가 아니다)
- [ ] 원본 SaaS의 브랜드·UI·카피가 복제되지 않았고, 로컬 실행 경로가
      명확하다
- [ ] (코드형·그래프 강화 번들만) §5 지식 그래프 검증 게이트를 통과했다
- [ ] `agent.md`에 학습 루프 강령(§2/§5.5 형식)이 존재하고, 위생 필터
      문구(PII·미공개 매출·원문 발췌 금지)가 실제로 있는지 확인했다
- [ ] `agent.md`가 번들에 없는 문서(예: 존재하지 않는 `design.md`, 문제
      상황 표 등)를 dangling 참조하지 않는다
- [ ] (코드형 번들만) core-공유 원칙을 지켰다(CLI/MCP 어댑터에 core에 없는
      신규 로직이 없다), 테스트가 그린이다
- [ ] 고위험 직무(법률·의료·금융·투자·채용)는 전문가 검토·규제 확인 문구를
      포함하고, 외부 지식은 원문이 아니라 출처만 표기했다
- [ ] 작성과 리뷰가 분리됐다 — 같은 컨텍스트 자기승인이 아니라 독립 검증
      게이트를 통과했다(§3)

## 7. 실제 실행으로 확인 (실행 시나리오 검증)

리뷰어는 최소 다음을 실제로 실행해 확인한다:

- CLI 서브커맨드가 문서와 1:1로 대응해 실제로 동작하는가
- MCP 서버가 시작 시 `agent.md` + 성장 레이어 병합본을 `instructions`로
  내보내는가
- 페르소나/기억 없이 시작했을 때 절대 규칙(예: "페르소나 없이 진단 시작
  금지")이 실제로 지켜지는가
- 위생 필터가 PII·미공개 매출 실측치를 넣었을 때 실제로 거부하는가

낯선(개발 환경이 아닌) 디렉터리에서 처음 설치하는 과정까지 한 번은
재현한다.
