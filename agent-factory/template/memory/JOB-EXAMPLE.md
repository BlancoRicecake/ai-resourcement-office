과거 작업 이력과 반응이 좋았던 패턴, 사용자 피드백에서 배운 점을 기록하는
파일이다.

## 기록 원칙

- 작업을 완료할 때마다 어떤 접근·구조·표현을 썼는지 남긴다.
- 사용자 피드백이나 실측 반응 데이터가 있으면 함께 기록한다.
- 반응 근거가 없는 패턴은 `검증 필요`로 표시하고 확정 패턴으로 올리지 않는다.

## 기록 형식

```md
## YYYY-MM-DD <작업 제목 또는 주제>

- <채울 자리: 직무별 핵심 항목 1>:
- <채울 자리: 직무별 핵심 항목 2>:
- 사용자 피드백:
- 반응(있으면):
- 배운 점:
- 다음 작업 반영:
```

## 유효한 패턴

- 아직 없음.

## 작업 기록

- 아직 기록된 작업 없음.

## 업데이트 규칙

기록은 자동 확정하지 않고 `<JOB>.md 후보`로 제안한 뒤 사용자 승인 후
반영한다.
