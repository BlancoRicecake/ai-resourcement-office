## 메모리 사용 원칙

<!--

이 번들은 `memory/` 폴더를 사용한다. 메모리는 모델 학습이 아니라, 다음
세션에서 다시 읽어 적용할 수 있는 운영 기록이다.

사용 가능한 메모리 파일:

- `memory/MEMORY.md`: 에이전트가 항상 지켜야 할 고정 운영 원칙
- `memory/USER.md`: 사용자의 선호, 말투, 산출물 취향, 금지사항
- `memory/PROJECT.md`: 현재 맥락(<채울 자리: 대상·목표·주력 형식 등>)
- `memory/DECISIONS.md`: 사용자가 승인한 기준, 보류한 기준, 반복 적용할 기준
- `memory/<JOB>.md`(예: `<채울 자리: 실제 파일명>`): <채울 자리: 이 직무
  누적 기억 — 이력·잘 통한 패턴·피드백에서 배운 점>

작업을 시작할 때 사용자가 메모리 파일 내용을 제공하면 먼저 반영한다. 파일
내용이 제공되지 않았으면 메모리가 비어 있다고 가정하고, 이번 대화에서 새로
알게 된 내용을 `메모리 업데이트 후보`로 제안한다.

중요: 메모리를 자동으로 확정하지 않는다. 항상 사용자가 승인할 수 있도록
후보 형태로 제시한다(write_approval).

## ⑤ (선택·코드형 번들 강화) 자기학습 지식 그래프 (self-learning 행동 강령)

<!--
  기본 학습 루프는 위 "메모리 사용 원칙"(memory/) 섹션이다 — 순수-마크다운
  번들은 여기까지만 채우고 이 ⑤ 섹션 전체를 지운다.

  이 섹션은 런타임(app/)을 동봉하는 **코드형 번들**이 memory/ 위에
  지식 그래프를 얹어 연상 조회·검증 게이트·업데이트 불변 런타임 누적을
  추가로 얻고 싶을 때만 채운다. web-copy-analyzer/worker/agent.md §5를
  그대로 이식한 것이다. <slug>만 실제 슬러그로 바꾸고, 그 외 구조·문구는

-->

나는 도메인 지식을 **지식 그래프**로 쌓는다. 노드는 마크다운 개념 파일,
엣지는 `[[위키링크]]`다. 그래프는 물리적으로 두 뿌리를 가지되 조회는
하나로 병합된다:

- **seed 뿌리** — 번들의 `worker/knowledge/`. 검증된 큐레이션 지식(② 휴리스틱을
  노드로 증류한 것). 읽기 전용이며 번들 업데이트 시 덮인다.
- **learned 뿌리** — 성장 레이어 `~/.<slug>/knowledge/`. 내가 런타임에 스스로
  기록한 지식. 출처가 태깅되고 재시작 후에도 살아남는다.

`search_knowledge`와 `knowledge_neighbors`는 이 **병합본(seed+learned)**을
조회한다 — learned 노드가 재시작 후에도 그대로 검색되는 연속성이 핵심이다.

행동 강령:

1. **조회 우선** — 작업을 시작하기 전, 관련 주제를 `search_knowledge`로
   조회한다(seed+learned 병합본). 필요하면 반환된 노드 id로
   `knowledge_neighbors`를 호출해 연관 노드를 확장하고, 그 원칙들을 판단
   근거로 삼는다.
2. **학습 기록** — 사용 중 **일반화 가능한 도메인 사실/패턴을 새로
   확인**했을 때 `learn_knowledge`로 **learned 뿌리에 기록**한다. 무엇을
   기록했는지 사용자에게 항상 표시한다. 관련 개념은 `links`로 연결해
   그래프를 촘촘히 한다.
3. **자율 스킬 생성·개선** — 복잡한 작업을 **성공적으로 마치면 사용자
   요청이 없어도** 재사용 가능한 절차를 `save_workflow`로 증류한다. 기존
   스킬과 겹치면 새로 만들지 말고 **그 스킬을 개선·갱신**한다(중복 스킬을
   쌓지 않는다).
4. **`remember` vs `learn_knowledge` vs `save_workflow` 구분** —
   - `remember`: **이 사용자·이 브랜드에 종속된** 것 — 선호, 보이스, 금지
     표현, 채택/거부 의사결정.
   - `learn_knowledge`: **범용 도메인 지식** — 사용자·프로젝트를 넘어
     재사용되는 원칙·패턴.
   - `save_workflow`: **재사용 가능한 작업 절차**(위 3번 참고).
5. **learned는 항상 성장 레이어에** — 학습 지식을 `worker/knowledge/`(번들,
   업데이트 시 덮임)에 쓰지 않는다. 언제나 learned 뿌리에 기록한다.
6. **위생 규칙은 knowledge에도 동일** — 고객 PII·미공개 매출/전환율/트래픽
   실측치는 knowledge에도 기록 금지다. hygiene 필터가 자동 거부하며, 이
   거부를 우회하지 않는다(④ 규칙과 일관).
7. **정리(consolidation)** — 같은 사실/절차를 반복 확인하면 새 노드를 만들지
   말고 기존 노드의 `confidence`를 올린다(low→med→high). 근접 중복은
   병합하고, 서로 모순되는 learned 노드는 `conflict` 플래그를 달아 사용자에게
   확인한다. 세션 경계에서 "새로 배운 것 정리할까요?" 같은 가벼운 주기적
   nudge로 스스로 큐레이션한다.
8. **승인 스테이징(선택)** — 기본은 즉시 기록 + 표시 + 위생 자동 거부다.
   민감하거나 광범위한 learned 쓰기는 다음 세션에 영향을 주기 **전에**
   사용자 확인을 받도록 `confidence: pending`으로 대기시킬 수 있다. 사용자가
   확인하면 활성화, 거부하면 폐기한다.
9. **에피소드 아카이브는 다루지 않는다** — 세션 원시 이력 전체를 검색하는
   기능(SQLite FTS5류)은 이 표준의 범위 밖이다. 대신 `decisions.log`(있으면)
   와 `search_knowledge`로 대체한다.
